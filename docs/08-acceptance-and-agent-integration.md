# 验收矩阵与企业 Agent 对接要求

## 1. 文档定位

本文档固定卓羽智能数据中台三期验收口径，并单独锁定企业内核级 agent 与平台对接的要求。

企业 agent 当前仍处于 MVP 阶段。一期平台必须先提供稳定、易读、可增量同步的数据契约，避免未来 agent 被平台业务表结构变化拖垮。

## 2. 一期验收矩阵：内部 BIM 资产管理试点

| 验收域 | 必测项 | 通过标准 | 阻塞级别 |
|---|---|---|---|
| 项目台账 | 导入不少于 20 个真实项目 | 项目编码、名称、阶段、专业、负责人、客户/业主可查 | P0 |
| 项目维护 | 后端 API 增删改查项目 | 普通维护人员可通过 REST API + Swagger/OpenAPI 完成建档、编辑、停用、筛选 | P0 |
| NAS 接管 | 扫描真实 NAS 目录 | 正确识别文件名、路径、格式、大小、项目归属置信度 | P0 |
| 文件登记 | 大文件不入库 | 数据库只存元数据、路径、checksum、状态 | P0 |
| 模型资源库 | 单项目多模型登记 | 单项目 50-100GB 量级模型元数据可管理 | P0 |
| 图纸资源库 | 二维图纸/CAD 元数据登记 | `.dwg/.dxf/.pdf` 默认按图纸治理，人工可修正 `.dwg` 类型 | P0 |
| 待审核队列 | 低置信扫描结果人工审核 | 可修正项目归属、文件类型、专业、版本后正式入库 | P0 |
| 搜索筛选 | 元数据 + 路径检索 | 可按项目名、项目编码、文件名、专业、阶段检索 | P0 |
| checksum | 重复/变更识别 | checksum 可异步补齐，重复文件可识别或标记 | P1 |
| 版本 | 企业内部版本规则 | 人工版本、文件名版本、默认 V1 规则明确 | P1 |
| 容量看板 | 10TB 量级元数据统计 | 项目数、模型数、总容量、专业分布、更新时间可查 | P1 |
| 下载 | 下载或打开入口 | 内部用户可通过平台或 agent 找到真实文件路径 | P0 |
| 权限 | 项目访问隔离 | 普通用户不能访问无权限项目资产 | P0 |
| 审计 | 关键动作和 agent 行为留痕 | 导入、扫描、审核、修改、删除申请、agent 查询和触发任务均可审计 | P0 |
| 任务 | 数据库任务表 + 应用内 worker | 扫描、checksum、隔离清理、永久删除任务可查看状态、进度、失败原因、重试结果 | P0 |
| 删除 | 受控物理删除 | 物理删除必须申请、审核、隔离 30 天、支持恢复，不能绕过审批直接删 NAS 文件 | P0 |
| Agent | 稳定读模型与 API Key 可用 | agent 可读 SQL View，可用项目范围 API Key 触发任务和提交申请，不依赖业务底表 | P0 |
| 增量同步 | 审计/事件流可用 | agent 可按事件 ID 或时间窗口拉取新增、修改、审核、删除和任务状态变化 | P0 |

## 3. 一期 MVP 场景

一期必须用真实或接近真实的数据完成以下闭环：

`20 个项目清单 -> NAS 路径映射 -> NAS 扫描任务 -> 高置信自动入库 -> 低置信待审核 -> 人工审核 -> 项目资产台账 -> 模型/图纸资源库 -> checksum 异步补齐 -> 容量看板 -> 企业 agent SQL View/API 查询 -> 事件流增量同步 -> 审计日志可查`

额外要求：

- 后续必须支持从 20 个项目扩展到公司全量几百个项目。
- 总规模按 `10TB` 文件资产元数据设计。
- 若 NAS 路径失效，平台必须能提示失效原因，不允许静默显示可用。
- 一期文件扫描白名单为 `.rvt`、`.dwg`、`.ifc`、`.nwd`、`.nwc`、`.dxf`、`.pdf`。
- `.rvt/.ifc/.nwd/.nwc` 默认归为模型，`.dwg/.dxf/.pdf` 默认归为图纸；`.dwg` 可人工审核改为模型。
- 项目归属只能由路径映射命中或路径/文件名精确包含项目编码自动确认，其余进入待审核。
- 一期人工审核和运维操作先通过 REST API + Swagger/OpenAPI 完成，前端页面不作为验收阻塞项。

## 3.1 真实 NAS 试点补充验收

真实 NAS 接入不改变一期架构和数据落点，只作为 Batch 1 数据底座后的结构适配与治理增强。当前真实 NAS 根路径固定为 `/Volumes/zyzn/卓羽智能项目`。

已确认的标准项目可进入正式资产库。`98`、`95`、`99` 等非标准目录、重复编号目录和疑似项目集合目录暂不导入数据库，等待企业 agent 接入后执行人工监控下的数据治理。

真实 NAS 试点必须满足：

| 验收项 | 通过标准 | 阻塞级别 |
|---|---|---|
| 一级目录发现 dry-run | 能输出项目编码建议、项目名称建议、路径映射建议、需人工确认目录、参考资料目录和忽略目录，不写正式资产 | P0 |
| 编码规则 | `数字-项目名` 目录优先取数字作为编码；重复数字前缀不能静默冲突，必须短哈希或人工确认 | P0 |
| 非项目目录治理 | 投标、参考、样板、标准、临时或专题资料目录不得自动成为正式项目 | P0 |
| 标准目录识别 | 能识别 `00_工作进度`、`01_文件收发`、`02_项目资源`、`03_过程文件`、`04_共享文件`、`05_发布文件`、`06_归档文件`、`07_浏览动画` 等二级目录线索 | P1 |
| 影子导入 | 只读扫描 NAS，不移动、不改名、不删除原文件 | P0 |
| 高价值目录加权 | `成果文件`、`发布文件`、`归档文件` 目录中的文件可提高归属置信度 | P1 |
| 临时目录降级 | `新建文件夹`、`临时文件`、`转换` 等目录中的文件默认进入待审核，不自动入库 | P0 |
| 低价值文件治理 | `.DS_Store`、`Thumbs.db`、`desktop.ini`、`~$` Office 锁文件、空文件不得自动入正式资产 | P0 |
| 扫描进度可见 | 大目录扫描任务必须返回当前进度、总量、百分比、最后扫描路径和跳过目录数量 | P0 |
| 扫描可取消与续扫 | 正在运行的扫描可取消；取消或失败后可续扫；续扫不得重复生成正式资产 | P0 |
| 扫描报告 | 扫描完成、失败或取消后可查询任务报告，报告至少包含扫描数、自动入库数、待审核数、失败数、跳过数和失败原因 | P0 |
| 文件类型扩展 | `DOC/DOCX`、`XLS/XLSX`、`PPT/PPTX`、`GLB/GLTF`、`ZIP/RAR`、`WPS` 可被分类为文档、表格、汇报、展示模型或归档包 | P1 |
| 非标准资料治理 | `98`、`95`、`99` 等目录先进入治理清单，不进入正式资产库；agent 给出拆分、编码、导入范围和风险建议后，人工确认才可入库 | P0 |
| 幂等重扫 | 同项目同 `storage_uri` 重扫只更新元数据和 `last_verified_at`，不重复插入正式资产 | P0 |
| 视图可查 | `FileAssetView` 能按项目、文件类型和 `source_type=NAS_SCAN` 查询真实 NAS 资产 | P0 |
| 现有脚本不回归 | `check-bim-asset-batch1.sh` 与 `check-scan-task-control.sh` 均通过 | P0 |

模拟验收可以在 `/tmp` 下构造标准项目、重复编号项目、项目集合目录、投标参考目录，等价验证编码、路径映射、分类、低价值文件、幂等重扫和非标准治理规则。真实 NAS 验收必须确保所有操作都是只读扫描。

## 4. 企业 Agent 对接契约

### 4.1 对接原则

- 企业 agent 权限很高，可以把数据库检索作为自身能力之一。
- 平台仍需提供稳定读模型，不能让 agent 直接耦合频繁变化的业务底表。
- 一期稳定读模型优先采用 MySQL SQL View。
- 当前一期 DB-2 首轮联调采用本机同环境：平台数据库运行在本机 Docker MySQL，企业 agent 后续更新到本地 Hermes 后也运行在同一环境。
- 当前不强制获取 shared-dev / staging 账号；只有多人远程协作、持续测试、演示或客户交付前类生产验证时才开通。
- agent REST/OpenAPI 接入采用 API Key，API Key 按项目范围授权，可授权全部项目或指定项目。
- 一期先满足 `元数据 + 路径` 检索。
- 文档正文检索、模型属性检索、构件级检索放到二期或三期。
- 平台保留 REST/OpenAPI，作为后续 agent 工具调用和客户交付接口。
- agent 可查询、触发 NAS 扫描/checksum 任务、提交标注或删除申请；正式数据变更必须人工审核。

### 4.3 REST/OpenAPI 保留接口

一期需保留以下 API 能力，后续企业 agent 可作为工具调用：

- 项目资产查询。
- 文件资产查询。
- 模型资产查询。
- 项目资产和容量统计必须支持资产来源过滤，例如 `assetSource=NAS_REAL*`。
- 文件资产查询必须提供分页接口，返回 `items/pageNo/pageSize/total`，并支持文件类型、专业、扩展名、关键字和资产来源过滤。
- 项目清单与 NAS 路径映射维护。
- CSV/xlsx 项目清单与路径映射导入。
- NAS 扫描任务查询。
- 待审核队列查询与审核。
- checksum 任务查询与重试。
- 容量统计查询。
- 文件访问或下载入口。
- agent 元数据搜索接口、任务触发接口和变更申请接口。
- 删除申请、审核、隔离、恢复和永久删除接口。
- 审计/事件流增量同步接口。

接口必须满足：

- 统一响应结构。
- 统一错误码。
- 响应包含 `traceId`。
- OpenAPI 文档可访问。
- 权限与审计策略明确。
- API Key 请求必须记录 agent 身份、项目授权范围、来源 IP、traceId、动作和结果。

### 4.4 增量同步要求

为避免 agent 每次全量扫描数据库，一期必须提供审计/事件流增量同步：

- 所有读模型仍应具备 `updated_at`，但增量同步以事件流为主。
- 逻辑删除、删除申请、审批、隔离、恢复和永久删除必须产生可识别事件。
- 扫描任务、导入任务、checksum 任务、文件变更必须产生审计或事件记录。
- agent 可按事件 ID 或时间窗口拉取变化数据。
- checksum 变化可作为文件内容变化的重要信号。

### 4.5 Agent 权限与删除安全

- API Key 必须绑定 agent 身份、状态、过期时间、最近使用时间和项目授权范围。
- agent 可读 SQL View 和只读 API，可触发扫描/checksum 任务，可提交标注或删除申请。
- agent 不允许直接修改正式资产记录，不允许绕过审核执行物理删除。
- 平台逻辑删除不触碰 NAS 原文件。
- NAS 原文件物理删除必须先提交申请，经人工审核通过后移动到隔离区。
- 隔离区保留期为 `30 天`，隔离期内必须支持恢复，超过隔离期后才允许受控永久删除。

### 4.6 一期落地前再次确认项

在企业 agent 接入前，主 agent 必须再次向企业 agent 开发负责人确认：

- 企业 agent 是否按当前裁决运行在本地 Hermes，与平台数据库处于同一台本机环境。
- 企业 agent 使用的数据库连接方式和只读账号策略；本机 DB-2 联调必须使用 `hermes_agent_ro`，不得使用平台应用主账号。
- 是否直接读 MySQL SQL View，还是读同步库、搜索引擎或向量库。
- 是否需要平台主动推送变更事件，或由 agent 定时拉取事件流。
- 是否需要 agent 读取真实 NAS 路径。
- 是否需要解析 PDF/Office 正文。
- 是否需要 agent 触发平台扫描任务。
- 是否需要 agent 调用平台下载接口。
- 是否已经触发 shared-dev / staging 需求；未触发时不等待远程环境，触发后必须由平台/运维按后置执行单提供 DSN 和只读账号。

### 4.7 当前已冻结的 DB-2 对接裁决

以下裁决用于当前一期 DB-2 联调和后续企业 agent 开发。除非主 agent、平台后端负责人、企业 agent 负责人和数据负责人共同更新本文档，否则不得在开发中自行改变。

| 问题 | 当前裁决 |
|---|---|
| Agent 是否直连 MySQL View | 是。一期 DB-2 首轮联调采用本机同环境，企业 agent 使用只读账号读取四个稳定 View，不允许读取业务底表。 |
| 是否通过同步库、搜索引擎或向量库读取 | 当前不走同步库、搜索引擎或向量库。一期只做 MySQL View 只读 proof-of-contract；后续若建设 mirror、搜索索引或向量索引，必须另行审批并执行脱敏、权限和持久化策略。 |
| 事件流是平台推送还是 agent 拉取 | 当前为 agent 拉取。主游标为 `AuditEventView.event_id`，`created_at` 只作为重叠窗口辅助字段；平台暂不主动推送。 |
| 权限字段如何传递 | 当前四个 View 没有稳定权限字段，Hermes mirror 层必须默认 `DENIED`。只有平台 API Key 项目授权范围或后续稳定权限字段证明可见时，资产才能进入 prompt、evidence 或 answer。 |
| 文件路径是否允许返回 | 本机一期内部联调允许返回真实 NAS 路径，仅限授权内部环境。shared-dev、staging、客户环境默认不直接暴露底层路径，应优先返回平台授权访问入口或脱敏路径。 |
| 真实项目名、文件名是否允许进入 agent memory | 不允许进入长期 memory、向量库、搜索库、外部日志、外部云服务或客户材料。本机联调可临时读取样例做字段握手，但不得持久化。 |
| 是否允许读取 PDF/Office 正文 | 当前一期不允许。PDF/Office 正文解析、OCR、向量化、全文搜索属于后续能力，必须单独定义权限、审计、脱敏和正文证据策略。 |
| 是否允许触发扫描任务 | 允许，但必须通过平台 API Key 和项目授权范围触发。agent 不得直写数据库，不得绕过平台创建任务。 |
| 是否允许调用下载接口 | 默认不直接放开真实下载。若需要调用，必须通过平台授权接口、项目权限校验和审计；客户环境不得默认返回真实 NAS 路径。 |
| shared-dev / staging 如何交付只读账号 | 当前不需要开通。触发多人远程协作、持续测试、演示或客户交付前验证后，由平台/运维创建等价只读账号，只授予四个 View 的 `SELECT`，密码通过安全渠道交付，不写入仓库，不复用本机 dev 密码。 |
| 客户环境如何执行脱敏和安全确认 | 客户环境默认 `STRUCTURE_ONLY`，先用 `SHOW COLUMNS` 或 `WHERE 1 = 0` 做字段握手。若需真实样例，必须业务负责人和数据负责人书面确认；否则由平台提供脱敏 View 或脱敏样本。 |

一句话边界：企业 agent 在一期只能把平台数据库当“只读资产目录”，不能把它当“可自由索引和长期记忆的知识库”。

## 5. 二期验收矩阵：客户交付完整版

| 验收域 | 必测项 | 通过标准 | 阻塞级别 |
|---|---|---|---|

## 6. 二期完整验收主链路

二期必须通过以下完整链路：

`客户环境部署 -> 初始化项目 -> 套用交付标准 -> 建立部位/专业/标段 -> 导入文档/图纸/模型 -> 模型轻量化 -> 构件解析 -> 文件和构件绑定 -> 提交审核 -> 整改闭环 -> 文档/图纸/模型交付视图 -> 项目驾驶舱 -> 报表导出 -> 审计追踪`

## 7. 三期验收原则

三期功能作为增值服务单独验收，不阻塞二期客户交付：

- 模型版本对比按项目或模型单独验收。
- 图模联动按图纸类型、构件映射质量和定位准确率验收。
- AI 审核按规则覆盖率、召回率、误报率和人工复核体验验收。
- 客户侧智能问答按资料召回、引用准确性、权限隔离和审计验收。

## 8. 缺陷分级与发布门槛

| 等级 | 定义 | 发布要求 |
|---|---|---|
| P0 | 数据丢失、权限泄漏、系统不可用、文件找不到、核心链路断裂 | 不允许发布 |
| P1 | 核心功能错误、预览不可用、审核流错误、接口不兼容 | 客户交付前必须清零 |
| P2 | 体验问题、边界错误、非核心统计偏差 | 可带计划发布，但必须记录 |
| P3 | 文案、样式、低频优化 | 可进入后续迭代 |

## 9. 交付文档验收

每期交付至少包含：

- PRD 与范围说明。
- 架构设计与接口契约。
- 数据库字段字典和迁移说明。
- 测试矩阵和验收报告。
- 部署和启动说明。
- 开发 agent prompt 和测试 agent prompt。
- 已知问题和后续计划。

二期客户版额外必须包含：

- 客户部署手册。
- 运维手册。
- 备份恢复手册。
- 用户操作手册。
- API 对接手册。
- 演示项目模板。
