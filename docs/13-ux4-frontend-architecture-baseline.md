# UX4 前端信息架构与交互基线

更新时间：2026-06-05

## 1. 文档定位

本文档是 `UX4：平台视觉与员工使用效率修复` 的前端基线。

它用于约束后续前端开发 agent：

- 先按本文件重构前端骨架、交互路径和页面层级。
- 不再按早期 MVP 页面继续堆功能卡片。
- 不新增后端业务能力。
- 不修改数据库、权限规则、对象存储规则或 BIM 引擎接口。
- 旧链接必须兼容跳转，不能因为重做 UI 导致历史链接、测试脚本或员工书签失效。

一句话目标：

> 员工进入卓羽智能数据中台后，能像使用成熟 SaaS 项目管理系统一样，先选项目，再围绕当前项目完成文件管理、工程主数据、交付闭环、BIM 协同和档案归档。

## 2. 参考图提炼

用户提供的参考图属于典型企业 SaaS 项目工作台，核心不是炫酷视觉，而是清晰的工作路径：

1. 左侧是稳定全局导航。
2. 顶部是项目上下文、项目切换、全局搜索、消息和用户信息。
3. 中央上方是项目身份区：项目图片、项目名称、编码、负责人、当前阶段、当前角色。
4. 项目内一级 tab 固定在项目身份区下方。
5. 页面主体按当前 tab 展示业务内容。
6. 右侧辅助栏用于进度、风险、快捷操作、最近动态。
7. 低频技术信息收起来，不放在首屏抢注意力。

参考图对本平台的启发：

- 不再用多个重复的“项目工作台导航”堆在页面里。
- 不再让全局菜单、项目菜单、页面按钮三套入口同时抢主视觉。
- 项目页必须先呈现“我在哪个项目、当前该做什么、下一步去哪”。
- 文件管理、工程主数据、交付闭环、BIM 协同、档案目录必须是同一项目下的几个工作区，不是互相割裂的功能页面。

## 3. UX4 设计原则

### 3.1 信息层级

前端只保留四层：

```text
全局壳层
└─ 项目启动台
   └─ 项目工作台
      └─ 项目内业务页面
```

解释：

- `全局壳层`：左侧导航、顶部栏、用户信息、全局搜索。
- `项目启动台`：项目列表、最近项目、待处理项目、对象化覆盖率概览。
- `项目工作台`：某一个项目的总入口。
- `项目内业务页面`：文件管理、工程主数据、交付闭环、BIM 协同等。

### 3.2 视觉重点

首屏必须优先回答三件事：

1. 当前项目是谁。
2. 当前项目处于什么状态。
3. 用户下一步最应该做什么。

禁止把以下内容放在首屏强展示：

- 平台内部 ID。
- `NAS_REAL_PILOT`、`ACTIVE` 等技术状态。
- checksum 细节。
- bucket、object key、storage uri。
- 长篇教程文字。
- 多个重复导航区。

这些内容可以进入：

- 项目详情抽屉。
- 技术信息折叠区。
- 文件详情侧栏。
- 管理员诊断区。

### 3.3 视觉风格

采用“精致企业 SaaS + 项目管理仪表盘”风格：

- 左侧深色品牌导航。
- 主内容浅色背景。
- 白色卡片、浅蓝高亮、细边框、柔和阴影。
- 关键状态使用蓝、绿、橙、红四类语义色。
- 表格保持企业中后台的高可读性。
- 少量轻玻璃效果只能用于项目身份卡、指标卡和右侧辅助面板。

禁止：

- 大面积毛玻璃导致文字不清。
- 大面积深色内容区。
- 营销风 hero。
- 过度装饰动画。
- 所有功能都做成同级卡片。

## 4. 全局壳层

### 4.1 左侧导航

左侧导航只放全局入口，不放项目内细碎功能。

建议结构：

```text
卓羽智能数据中台
├─ 项目启动台
├─ 资产治理
├─ BIM 协同
├─ 管理中心
│
├─ 我的项目
│  ├─ 最近项目 1
│  ├─ 最近项目 2
│  └─ 添加/搜索项目
│
├─ 帮助中心
└─ Hermes 助手
```

对应规则：

- `项目启动台` 是默认入口，路径仍为 `/data-steward/assets`。
- `资产治理` 进入全局资产治理总览，不直接展开扫描、质量、非标目录等低频子项。
- `BIM 协同` 进入综合 BIM 协同页。
- `管理中心` 只对管理员展示。
- `我的项目` 用于快速进入最近项目，不替代项目启动台。
- 项目内的“文件管理、工程主数据、交付闭环”等不再放在左侧一级菜单。

### 4.2 顶部栏

顶部栏承担全局操作：

```text
面包屑 / 当前项目选择 / 全局搜索 / 通知 / 帮助 / 用户
```

顶部栏行为：

- 在项目外页面显示：`项目启动台 > 当前页面`。
- 在项目内页面显示：`项目启动台 > 项目名称 > 当前 tab`。
- 返回资产总览按钮放在顶部栏左侧，替代“当前项目工作台”胶囊。
- 项目切换器放在顶部中间或右侧，显示项目名称 + 项目编码。
- 全局搜索可以搜索文件、对象、任务，但 UX4 只做前端入口，不新增搜索后端能力。

## 5. 项目启动台

路径：

```text
/data-steward/assets
```

定位：

> 选择项目的地方，不是展示所有技术统计的地方。

首屏结构：

```text
项目启动台
├─ 推荐进入项目
├─ 搜索项目
├─ 真实项目列表
├─ 最近项目
└─ 需要处理的项目
```

项目卡片展示字段：

- 项目名称。
- 项目编码。
- 项目来源：真实 NAS / 样例 / 测试 / 归档。
- 接入状态：未接入 / 已登记资产 / 已确认主数据 / 交付治理中。
- 文件数。
- 对象化状态：已完成 / 部分完成 / 历史 NAS 链路。
- 关键风险数。

默认隐藏：

- 内部项目 ID。
- 原始路径。
- 复杂任务编号。
- 过多统计图表。

主要跳转：

```text
点击项目卡片 -> /data-steward/assets/:projectId
点击文件管理 -> /data-steward/assets/:projectId?tab=files
点击项目可视化 -> /data-steward/assets/:projectId?tab=dashboard
点击 BIM 协同 -> /bim-collaboration?projectId=:projectId
```

## 6. 项目工作台

路径：

```text
/data-steward/assets/:projectId
```

定位：

> 某一个项目的一站式工作入口。

### 6.1 项目身份区

项目身份区参考图中项目头部，必须展示：

- 项目图片或默认建筑封面。
- 项目名称。
- 项目编码。
- 负责人。
- 当前角色。
- 当前阶段。
- 当前主状态。

状态示例：

- `工程主数据确认中`
- `文件对象化完成`
- `交付治理中`
- `BIM Viewer 可用`

技术信息折叠：

- 平台内部 ID。
- assetSource。
- assetStatus。
- object storage provider。
- 当前对象化覆盖率详情。

### 6.2 项目内主 tab

项目内一级 tab 固定为：

```text
概览
文件管理
工程主数据
交付闭环
BIM 协同
档案目录
```

对应路径或状态：

```text
概览       -> /data-steward/assets/:projectId?tab=dashboard
文件管理   -> /data-steward/assets/:projectId?tab=files
工程主数据 -> /data-steward/assets/:projectId?tab=ownership 或项目内 master-data 页面
交付闭环   -> /data-steward/assets/:projectId/work/document-delivery
BIM 协同   -> /bim-collaboration?projectId=:projectId
档案目录   -> /data-steward/assets/:projectId/work/delivery-package
```

注意：

- 一级 tab 不超过 6 个。
- 低频功能进入 tab 内二级按钮或“更多操作”。
- 不再在项目详情页顶部重复放一套 `ProjectWorkspaceNav` 大卡片。

### 6.3 右侧辅助栏

项目工作台可以使用右侧辅助栏，展示：

- 项目健康度。
- 风险提醒。
- 最近动态。
- 快捷操作。
- Viewer 状态。

右侧辅助栏不放复杂表格。

## 7. 概览页

定位：

> 告诉用户这个项目现在处于哪个阶段，下一步该做什么。

布局：

```text
项目阶段条
├─ 资产接入
├─ 工程主数据
├─ 交付闭环
├─ BIM 协同
└─ 档案目录

今日建议 / 下一步行动
核心卡片：文件管理 / 工程主数据 / 交付状态
最近文件 / 最近处理记录
右侧：项目健康度 / 风险提醒 / 最近动态 / Viewer 状态
```

交互：

- 点击 `文件管理` 进入文件管理 tab。
- 点击 `确认主数据` 进入工程主数据 tab。
- 点击 `进入交付中心` 进入交付闭环。
- 点击 `打开 Viewer` 进入 BIM 协同。

## 8. 文件管理页

定位：

> 平台最核心、最常用的文件管理器。

参考图 4 的文件管理结构：

```text
二级 tab：全部文件 / 目录浏览 / 待归属 / 对象存储 / 质量异常
筛选栏：文件名、类型、专业、阶段、时间、更多筛选
工具栏：下载、预览、移动、复制、重命名、版本、归属、更多
左侧：目录树
中间：文件表
右侧：文件详情
```

默认表格字段：

- 文件名。
- 版本。
- 文件类型。
- 专业。
- 归属节点。
- 存储状态。
- 预览状态。
- 大小。
- 更新时间。

默认隐藏字段：

- 内部 fileId。
- checksum。
- objectVersion。
- bucket / objectKey。
- storage_uri。
- 真实 NAS 路径。

交互规则：

- 左键单选。
- Ctrl / Command 多选。
- Shift 连选。
- 右键打开上下文菜单。
- 双击文件夹进入目录。
- 双击 PDF / 图片打开受控预览。
- 双击 BIM 模型进入轻量化 Viewer 或任务状态。
- 搜索时默认在整个项目范围搜索，不只搜当前目录。
- 清空搜索后恢复当前目录浏览。

跳转规则：

```text
文件详情 -> 右侧详情栏，不跳离页面
所属工程节点 -> 工程主数据 tab 并定位节点
BIM 模型预览 -> /bim-collaboration?projectId=:projectId&preview=glandar
对象存储状态 -> 文件服务页或对象存储二级 tab
```

## 9. 工程主数据页

定位：

> 先定义规则，再按规则交付。

结构参考图 3：

```text
工程主数据完成度
步骤条：确认部位树 -> 锁定节点类型 -> 配置交付物标准 -> 生成规则草案
问题提醒
部位树预览
节点字段/规则表
右侧：覆盖率、最近更新、相关模板
```

二级 tab：

```text
接入向导
部位树
节点类型
交付物标准
工程树可视化
```

交互：

- 未确认主数据时，交付闭环 tab 可以查看状态，但写操作提示“请先确认主数据”。
- 工程树节点点击后显示该节点文件。
- 文件可以反查所属工程节点。
- 模型/图纸缺口分析必须标注“基于目录和文件名线索，不是 BIM 构件解析”。

跳转：

```text
确认部位树 -> project-master-data-sections
锁定节点类型 -> project-master-data-node-types
配置交付物标准 -> project-master-data-deliverable-standard
查看节点文件 -> 项目详情 tab=ownership 并定位节点
```

## 10. 交付闭环页

定位：

> 让员工知道缺什么、补什么、谁审核、能不能交付。

一级 tab 下的二级 tab：

```text
交付状态
文档交付
图纸交付
整改闭环
交付包草案
```

页面结构参考图 1/2：

```text
下一步行动
状态卡：待补交 / 待审核 / 已驳回 / 已通过
待交付清单
右侧：审核进度 / 风险提醒 / 快捷操作
```

交互：

- `选择文件补交` 调用现有批量挂接。
- `提交审核` 仍需人工确认。
- `整改记录` 进入整改闭环。
- `交付包草案` 只做 dry-run 或受控草案，不自动生成真实 ZIP。

## 11. BIM 协同页

定位：

> 项目级 BIM 协同和 Viewer 入口，不替代文件管理。

入口：

```text
/bim-collaboration?projectId=:projectId
```

结构：

- 默认首屏是综合驾驶舱。
- 资产文件、模型文件、图纸文件、管理对象、质量风险、交付完成率作为驾驶舱指标。
- READY 模型列表分页展示。
- Viewer 只在 READY 时显示“打开 Viewer”。
- 构件拾取、属性、爆炸功能只在真实 Viewer 中展示，不在项目概览夸大。

边界文案：

- 可以说：`模型已轻量化，可通过 Viewer 预览`。
- 不能说：`平台已完成 BIM 构件级智能理解`。

## 12. 档案目录页

定位：

> 从交付闭环结果生成可审计、可追溯的档案目录。

结构：

```text
交付包草案列表
草案详情
档案目录树
导出清单
预检查结果
```

交互：

- `生成草案` 不复制真实文件。
- `导出 manifest` 只导出清单。
- 后续真实打包必须单独批次确认。

## 13. 旧链接兼容

旧链接不得删除，必须跳转到项目上下文内。

兼容规则：

```text
/master-data/sections
  -> /data-steward/assets/:currentProjectId/master-data/sections

/master-data/node-types
  -> /data-steward/assets/:currentProjectId/master-data/node-types

/master-data/deliverable-standard
  -> /data-steward/assets/:currentProjectId/master-data/deliverable-standard

/work/document-delivery
  -> /data-steward/assets/:currentProjectId/work/document-delivery

/work/drawing-delivery
  -> /data-steward/assets/:currentProjectId/work/drawing-delivery

/work/rectifications
  -> /data-steward/assets/:currentProjectId/work/rectifications

/data-steward/models
  -> /data-steward/assets/:currentProjectId/data-steward/models

/data-steward/objects
  -> /data-steward/assets/:currentProjectId/data-steward/objects
```

如果没有 currentProjectId：

- 跳到 `/data-steward/assets`。
- 页面提示“请先选择项目”。

## 14. Hermes 入口

Hermes 仍作为辅助，不作为主导航。

显示规则：

- 项目启动台：可问“哪些项目需要处理”。
- 项目工作台：可问“当前项目下一步做什么”。
- 文件管理：可问“这个文件可能属于哪里”。
- 工程主数据：可问“当前工程树缺什么”。
- 交付闭环：可问“还缺哪些交付资料”。

边界：

- Hermes 不自动写库。
- Hermes 不自动挂接文件。
- Hermes 不读取正文，除非后续 M4/M5 evidence 能力完成。
- Hermes 回答必须显示证据边界。

## 15. UX4 开发分批

### UX4-A：壳层和路由基线

交付：

- 重构左侧导航。
- 重构顶部栏。
- 移除重复项目工作台导航。
- 固定项目内一级 tab。
- 保留旧链接兼容。

### UX4-B：项目启动台

交付：

- 项目卡片重排。
- 搜索、最近项目、待处理项目前置。
- 隐藏技术字段。

### UX4-C：项目工作台概览

交付：

- 项目身份区。
- 项目阶段条。
- 今日建议。
- 核心入口卡片。
- 右侧辅助栏。

### UX4-D：文件管理器

交付：

- 文件管理布局按参考图重构。
- 左目录、中表格、右详情。
- 全项目搜索和目录浏览口径清晰。
- 右键、多选、双击、预览、归属、对象存储状态入口清晰。

### UX4-E：工程主数据和交付闭环

交付：

- 工程主数据步骤化。
- 交付闭环状态卡和待交付清单重构。
- 工程树、缺失项、补交、审核关系清楚。

### UX4-F：BIM 协同和档案目录收口

交付：

- BIM 协同默认综合驾驶舱。
- READY Viewer 状态清晰。
- 档案目录和交付包草案页面风格统一。

## 16. UX4 测试标准

必测：

- 登录后进入项目启动台。
- 选择 105 / 503 进入项目工作台。
- 顶部项目上下文、项目 tab、返回按钮正确。
- 文件管理目录浏览可用。
- 文件管理全项目搜索可用。
- 文件详情不泄露真实 NAS 路径、bucket、object key。
- 工程主数据页可理解当前步骤和缺口。
- 文档/图纸交付可看懂缺失、补交、审核、整改。
- BIM 协同 READY 模型可打开 Viewer。
- 旧链接兼容跳转不白屏。
- 1280 / 1440 / 1920 宽度无横向溢出。

禁止通过：

- 修改后端业务逻辑。
- 删除旧路由。
- 页面泄露真实 NAS 路径或对象存储 key。
- 把 Hermes 或 BIM 能力夸大为已完成智能理解。
- 页面首屏继续堆大量技术字段。

## 17. 对开发 agent 的硬边界

UX4 开发 agent 只能修改：

- `frontend/**`
- `handoff/dev-agent/latest-report.md`

如确实需要修改 `docs/**` 或 `backend/**`：

- 必须停止。
- 写明原因。
- 等主 agent 单独批准。

UX4 不允许：

- 新增后端接口。
- 修改数据库迁移。
- 改权限规则。
- 改对象存储读取规则。
- 改葛兰岱尔 Station 业务接口。
- 新增 Hermes evidence 能力。

## 18. 基线结论

UX4 的成功标准不是“更好看一点”，而是让平台形成稳定、成熟、可培训的前端工作路径：

```text
选择项目
-> 看项目状态
-> 管文件
-> 确认工程主数据
-> 做交付闭环
-> 看 BIM 协同
-> 出档案目录
```

后续所有前端开发必须围绕这条路径服务。
