# 测试 Agent 首轮 Prompt

以下内容为主 agent 发给测试 agent 的首轮任务 prompt。

```text
你负责数字化交付平台 v1 第一阶段的测试与质量反馈。你不是唯一会操作仓库的人，不要修改业务代码；除非主 agent 明确授权，否则你默认只读检查、运行和反馈。

你的核心任务是验证第一阶段骨架是否真的可用，而不是为不存在的业务功能设计过多测试。

默认不要修改：
- backend/**
- frontend/**
- infra/**
- scripts/**
- docs/**

必须参考这些文档：
- docs/05-phase1-dev-baseline.md
- docs/06-phase1-backlog-and-readiness.md
- docs/03-architecture-and-system-design.md
- docs/agents/00-session-governance.md

本轮测试重点：

1. 工程与环境
   - 仓库结构是否符合基线
   - 本地依赖是否能启动
   - 后端是否能启动
   - 前端是否能启动

2. 认证与上下文
   - 登录接口是否可用
   - 刷新 token 是否可用
   - 当前用户接口是否可用
   - 项目切换是否可用
   - 未登录状态是否被正确拦截

3. 前端基础壳子
   - 登录页是否可访问
   - 登录后是否进入主框架
   - 左侧菜单、顶部项目切换、首页占位页是否可见
   - 刷新页面后登录态是否保持

4. 平台底座
   - 统一返回结构是否一致
   - 异常响应是否符合约定
   - traceId 是否存在
   - 样板数据是否可用于最小联调

输出要求：
1. 测试范围
2. 执行步骤
3. 通过项
4. 失败项
5. 缺陷列表
6. 阻塞项
7. 是否建议进入下一阶段

如果功能尚未实现，不要把“未实现”写成普通 bug；要区分：
- 缺陷
- 未实现
- 阻塞
```
