## 2. 执行步骤

1. 准备执行环境：`Java 21`、`Node 20 LTS`、`pnpm`、Docker。
2. 启动基础依赖：MySQL、Redis、MinIO；执行数据库迁移与初始化样板数据。
3. 静态核对工程骨架：检查后端模块、前端模块目录、`infra/` 与 `scripts/` 是否齐全。
4. 启动后端，确认应用无致命启动错误，访问 OpenAPI 页面。
5. 启动前端，确认登录页可访问。
6. 执行认证链路冒烟：
   - 登录
   - 刷新 token
   - 获取当前用户
   - 切换项目
   - 未登录访问受保护接口
7. 执行前端壳子冒烟：
   - 登录进入主框架
   - 查看左侧菜单、顶部项目切换、首页占位页
   - 刷新浏览器，确认登录态与当前项目仍在
8. 抽查底座契约：
   - 成功接口响应结构
   - 失败接口响应结构
   - `traceId`
   - 分页结构
   - 错误码格式
9. 用样板账号进入样板项目，确认最小联调链路可跑通。
10. 抽查审计日志，至少验证登录、项目切换或其他关键动作具备审计记录。

## 3. 通过判定

| 验收项 | 通过判定 |
|---|---|
| 工程骨架 | 目录与模块结构符合基线文档，且前后端工程均可独立启动 |
| 本地依赖 | MySQL、Redis、MinIO 均健康可连接，初始化后无关键报错 |
| 后端启动 | 应用可启动并对外提供 API，OpenAPI 页面可访问，Flyway 执行成功 |
| 前端启动 | 登录页正常渲染，无阻塞性白屏或启动报错 |
| 登录 | 返回可用的 access/refresh 凭证，后续受保护接口可访问 |
| Refresh Token | 能获取新的可用凭证，旧会话逻辑不异常 |
| 当前用户 | 返回当前用户身份信息，且包含可用项目上下文 |
| 项目切换 | 切换后后续接口和前端状态使用新的 `project_id` 上下文 |
| 未登录拦截 | 未携带登录凭证访问受保护资源时，被后端拒绝且前端正确跳转或提示 |
| 前端主框架 | 登录后可见主布局，左侧菜单和顶部项目切换正常显示 |
| 登录态保持 | 浏览器刷新后无需重新登录，当前项目状态不丢失 |
| 统一返回结构 | 成功/失败响应均包含 `code/message/data/traceId/timestamp` |
| 错误码规范 | 失败码采用 `{DOMAIN}_{SCENARIO}` 风格 |
| 分页结构 | 分页接口返回 `items/pageNo/pageSize/total` |
| traceId | 每次请求响应都带 `traceId`，便于追踪 |
| AuditLog | 关键动作产生审计记录，至少可查到动作、操作者、时间、项目 |
| 样板数据 | 新环境下可直接登录并进入样板项目，菜单权限足以完成最小联调 |

## 4. 潜在阻塞

| 阻塞项 | 影响 |
|---|---|
| `docker-compose` 或依赖服务不可用 | 无法完成后端启动、登录链路、样板联调 |
| Flyway 未落地或迁移失败 | 数据库结构不可用，后续接口验证全部失真 |
| 样板账号、样板项目、最小权限未初始化 | 无法验证登录后主链路与项目上下文 |
| 登录、刷新、当前用户、项目切换任一接口缺失 | 第一阶段底座验收不能通过 |
| 统一响应或 `traceId` 未全局生效 | 平台底座契约未达标，阻塞后续联调 |
| 前后端接口契约不一致 | 登录后无法进入主框架，或项目切换状态失真 |
| RBAC 或菜单最小集缺失 | 无法验证前端壳子与权限控制链路 |

## 5. 回归优先级

| 优先级 | 回归重点 | 触发时机 |
|---|---|---|
| P0 | 启动链路、登录、刷新 token、当前用户、项目切换、未登录拦截、统一返回、`traceId`、样板数据可登录 | 任一 `core`、认证、拦截器、网关、状态管理改动后立即回归 |
| P1 | 前端主框架、菜单权限、首页占位页、登录态保持、分页结构、`AuditLog` | 任一前端壳子、RBAC、审计、分页 DTO 改动后回归 |
| P2 | OpenAPI 可访问性、错误码文案一致性、空状态和前置条件提示组件 | 文档页、异常处理、通用组件调整后回归 |

## 6. 对开发交付的额外要求

为了让测试能立即落地，开发 agent 每次交付必须附：

1. 启动命令
2. 样板账号和密码
3. 样板项目编码和名称
4. 已实现接口清单
