# 数字化交付平台 v1 PRD

> 版本说明：本文档保留早期 `标准驱动交付 MVP` 的产品口径和链路说明。当前项目已经升级为三期路线，后续开发必须同时参考 [07-complete-delivery-prd.md](/Users/vc/Documents/数字化交付平台/docs/07-complete-delivery-prd.md) 和 [08-acceptance-and-agent-integration.md](/Users/vc/Documents/数字化交付平台/docs/08-acceptance-and-agent-integration.md)。一期开发优先级已调整为 `后端数据治理优先 + 内部 BIM 资产管理试点 + NAS 原地接管 + 企业 agent 可检索底座`，二期继续完成客户可交付的数字化交付平台完整版。

## 1. 项目背景

公司现有售前材料、标准方案和竞品拆解都指向同一个结论：客户真正要的不是一个资料库，而是一套能把工程交付资料沉淀成数字资产，并继续服务运维接管和业务协同的平台。

现有平台的共性问题有两个：

- 成本高，客户受制于供应商。
- 虽然能力全，但交付标准与实施路径不够透明，用户容易在前置配置阶段迷失。

因此，v1 的目标不是复制一个更大的杂合平台，而是做出一套更清晰、更可控、以建筑机电/BIM 交付为核心的数字化交付底座。

## 2. 产品定位

数字化交付平台 v1 是一套面向 `建筑机电/BIM交付` 场景的私有化平台，帮助项目从 `部位/EBS策划`、`交付标准定义`、`文件与模型归集`、`管理对象绑定` 走到 `文档/图纸交付视图` 和 `项目可视化展示`。

平台不以三维浏览为主卖点，而以 `标准驱动交付` 为核心竞争力。

## 3. 产品目标

### 3.1 v1 业务目标

- 建立一套可执行的项目交付标准底座。
- 打通文档、图纸、模型三类资源的统一归集和加工链路。
- 把交付资源绑定到工程部位和管理对象，形成可追踪的数字资产关系。
- 为文档交付视图、图纸交付视图、项目首页和可视化大屏提供统一数据底座。

### 3.2 v1 组织目标

- 让内部开发团队不再依赖外部高价平台。
- 让售前、实施、交付、运维四类角色使用同一套术语和主数据模型。
- 为后续 AI 审核、OCR、知识检索、智能问答预留结构化扩展点。

## 4. 目标用户

### 4.1 平台管理员

- 维护租户、组织、项目、角色、模板、字典和审计策略。

### 4.2 项目管理员

- 建立项目、配置部位树、锁定节点类型、维护交付物标准、控制项目级权限。

### 4.3 交付工程师

- 上传文件、处理模型、创建集成、维护管理对象、执行文档与图纸交付绑定。

### 4.4 审核人

- 查看交付标准、检查资源绑定完整性、确认交付链路是否可进入下游视图。

### 4.5 只读查看者

- 浏览项目首页、文档/图纸交付视图、大屏和三维工作台入口。

## 6. 核心业务流程

### 6.1 标准驱动交付主链路

1. 新建项目
2. 建立工程管理部位树
3. 配置节点类型
4. 锁定节点类型
5. 定义交付物
6. 配置交付物类型和属性
7. 导入或建立文档/图纸目录模板
8. 上传文档、图纸、模型文件
9. 完成模型集成
10. 生成或维护管理对象
11. 将资源与部位、对象、交付物标准建立绑定
12. 打开文档交付视图或图纸交付视图
13. 在项目首页、大屏、3D 工作台消费交付结果

### 6.2 平台设计要求

- 任意下游页面都必须能判断自己的前置条件是否满足。
- 前置条件不满足时，页面必须给出缺失项列表和修复入口。
- 所有关键动作必须留下审计日志。

## 7. 功能需求

## 7.1 平台与权限

### 7.1.1 项目管理

- 支持创建、编辑、归档项目。
- 支持项目基本信息、行业标签、建设单位、施工单位、设计单位、监理单位等字段。
- 支持项目级启停控制。

### 7.1.2 组织与角色

- 支持组织树和用户归属管理。
- 支持角色模板：平台管理员、项目管理员、交付工程师、审核人、只读查看者。
- 支持按项目授予角色。

### 7.1.3 消息与审计

- 支持站内消息计数、已读未读状态。
- 支持对关键动作记录时间、用户、对象、前后值摘要。

## 7.2 主数据与标准

### 7.2.1 工程管理部位树

- 支持项目级部位树创建与维护。
- 支持树节点编码、名称、层级、父子关系。
- 支持后续资源绑定引用。

### 7.2.2 节点类型锁定

- 支持项目管理员配置节点类型。
- 支持节点类型锁定动作。
- 锁定前，下游交付物定义、类型和属性配置不可进入正式可用状态。

### 7.2.3 交付物标准

- 支持交付物定义、交付物类型设置、交付物属性设置。
- 支持标准模板化和项目实例化。
- 支持属性单位、示例值、必填规则、枚举值说明。

### 7.2.4 目录模板

- 支持文档目录模板。
- 支持图纸目录模板。
- 支持项目引用模板后再按项目微调。

### 7.2.5 关联模型

- 支持交付物标准与模型资源的绑定。
- 支持自动关联和人工修正。

## 7.3 数据管家

### 7.3.1 文件资源管理

- 支持文档、图纸、模型三类文件上传。
- 支持文件夹、新建目录、目录模板引入。
- 支持文件处理状态、版本、耗时、创建人、创建时间展示。

### 7.3.2 模型集成

- 支持从模型文件创建集成。
- 支持记录文件数量、总大小、集成状态、处理耗时。
- 支持发布后供三维适配层消费。

### 7.3.3 管理对象

- 支持将模型集成或资源文件抽象为管理对象。
- 支持启用/停用。
- 支持后续与部位、交付物、视图绑定。

### 7.3.4 文件服务

- 支持将资源暴露为稳定服务入口。
- 支持服务状态管理。

## 7.4 工作与可视化

### 7.4.1 项目首页

- 展示待办、消息、分享、回收站。
- 展示项目数据概览：
  - 3D 模型数/总大小
  - 地形数/总大小
  - 图纸文档数/总大小
  - 其他文档数/总大小
  - 集成数

### 7.4.2 文档交付管理

- 支持文档目录浏览与维护。
- 支持文件挂接到目录、部位或管理对象。
- 支持搜索和状态筛选。

### 7.4.3 文档交付视图

- 支持按工程管理部位查看交付结果。
- 支持查看资源详情和关联关系。
- 前置条件不满足时，必须提示缺失项。

### 7.4.4 图纸交付管理与视图

- 与文档交付同构。
- 与文档交付共享同一套部位树、交付物标准和审计体系。

### 7.4.5 看板与三维入口

- 支持文件看板。
- 支持智慧大屏入口。
- 支持三维工作台入口。
- 不要求 v1 完成全部 3D 业务逻辑，但必须完成适配接口预留。

## 8. 非功能需求

- 私有化部署优先
- 权限隔离明确
- 审计可追踪
- 关键页面具备前置条件校验
- 文件与模型处理具备异步状态
- 交付标准与项目数据解耦，可模板化复用

## 9. 成功标准

- 项目团队可以从零创建一个样板项目，并完整跑通交付标准链路。
- 文档、图纸、模型可统一归集，并形成管理对象。
- 文档交付和图纸交付视图可以基于标准和部位树打开。
- 大屏和三维入口能拿到项目上下文。
- 平台关键动作都有审计记录。

## 10. 验收标准

### 10.1 主数据链路

- 新建项目成功
- 部位树建立成功
- 节点类型配置并锁定成功
- 交付物定义、类型、属性配置成功

### 10.2 数据管家链路

- 上传 `文档 + 图纸 + 模型` 三类文件成功
- 模型集成成功
- 管理对象创建成功
- 文件与管理对象关系可查询

### 10.3 交付链路

- 文档交付目录创建成功
- 图纸交付目录创建成功
- 文件可挂接到部位或管理对象
- 文档交付视图可打开
- 图纸交付视图可打开

### 10.4 可视化链路

- 项目上下文可传入大屏
- 项目上下文可传入 3D 适配层
- 关键动作可在审计日志中检索

### 10.5 MVP 场景

必须完成以下完整闭环：

`样板项目 -> 一个部位树 -> 一套节点类型 -> 一组交付物标准 -> 一份模型集成 -> 一组管理对象 -> 一条文档交付视图`
