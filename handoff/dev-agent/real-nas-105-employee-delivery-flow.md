## 3. 员工操作总流程

员工视角的实际顺序是：

1. 进入项目资产，先确认真实项目文件是否已接入。
2. 进入初始化向导，查看目录证据和草案风险。
3. 预览并确认应用工程主数据草案。
4. 查看部位树、节点类型、交付物标准，确认交付规则已经生成。
5. 进入图纸交付，查看缺失项。
6. 从缺失项选择文件补交。
7. 从项目资产目录中搜索并选择真实文件。
8. 保存为草稿。
9. 提交审核。
10. 运行导出预检查，查看还差什么。
11. 进入交付治理助手，查看缺失解释和推荐挂接方案。
12. 回到文件管理，核对平台文件 ID、目录位置和文件元数据。

## 9. 我作为员工的体验判断

整体结论：

当前流程已经能把“真实资产目录 -> 工程主数据草案 -> 图纸缺失项 -> 文件挂接 -> 提交审核 -> 预检查”串起来，作为演示和内部试用是可跑通的。

但如果让不懂业务的普通员工直接使用，仍然会有明显风险。平台目前能告诉员工“下一步做什么”，但还不能保证员工知道“为什么这么做、选哪个文件才对、当前工程结构是否真实”。

### 9.1 需要优先修的问题

1. 模板部位树过于粗糙

当前生成的是“项目 / 单体 / 楼层 / 系统”四层骨架，不是启航华居真实楼栋、楼层、地下室、机房、专业系统结构。员工照着这个补交，会把真实文件挂到抽象节点上，后续交付价值有限。

建议：把“应用模板草案”升级为“Agent 基于真实目录线索生成工程主数据草案，负责人审批后落库”。模板只能作为兜底，不应成为真实项目默认主路径。

2. 工作中心存在状态提示不一致

在图纸交付、文档交付、整改闭环页面中，底部曾仍显示“请先生成 / 确认工程主数据草案；工作中心页面会保留阻塞提示”，但页面主体又显示主数据已就绪并允许交付。回到项目资产页后导航恢复为“主数据已就绪”。

建议：修复项目工作台导航和工作中心提示的状态刷新逻辑，避免员工以为当前流程仍被阻塞。

3. 文档交付 `0/0 = 100%` 容易误导

文档交付页显示应交 0、完整率 100%，但交付物标准里确实有 PDF 文档和 Office 文档类型。普通员工会误以为文档交付已经完成。

建议：当应交为 0 时，不显示“100%完成”作为主结论，而显示“当前规则未生成文档应交项，请负责人确认标准是否覆盖文档交付”。

4. 推荐挂接需要收紧文件类型匹配

交付治理助手生成图纸推荐时，观察到“PDF 图纸”候选里出现 DWG 文件。虽然需要人工确认才会挂接，但对不懂业务的员工来说容易误选。

建议：推荐策略至少按交付类型、文件类型、扩展名做硬过滤；跨类型推荐必须降级为低置信并强提示原因。

5. 审核角色与员工角色边界不清

我使用的是 `platform.admin`，所以员工提交后页面直接出现“通过 / 驳回”。真实员工不应该在自己提交后立即拥有审核动作。

建议：用真实员工账号、审核人账号分别跑权限验收；普通员工只看到提交、记录、整改处理，不看到审核通过/驳回。

### 9.2 可以后置但应纳入路线的问题

1. DWG 在线预览仍依赖 CAD 转换

当前预检查能正确提示“需 CAD 转换”，也说明原始文件仍可按权限交付或下载。这个提示是清楚的，但后续如果希望员工在线核图，需要接入 CAD 转换/查看引擎。

2. 真实项目导入需要项目级重置/演练工具

本次清空通过数据库完成，不适合交给业务人员。后续可以做一个受控的“项目演练重置”或“重新生成主数据草案”入口，只清平台治理数据，不碰 NAS 文件。

3. 文件管理体验已有进步，但还可以继续贴近桌面文件管理器

当前能记住本项目位置，能看到平台文件 ID 和目录树。后续仍建议继续补：

- 列表/网格/紧凑视图切换
- 不同扩展名图标
- 最近访问目录
- 收藏目录
- 面包屑更强的回退能力

## 10. 给主线开发的建议优先级

P0：

- 不建议再让真实项目直接依赖模板部位树进入正式交付。必须尽快插入“Agent 生成真实工程主数据草案 + 人工审批”能力，否则真实项目会被抽象节点架空。

P1：

- 修复工作中心“主数据已就绪/请先确认主数据”的提示不一致。
- 修复文档交付 `0/0=100%` 的误导性表达。
- 收紧交付治理助手推荐的文件类型匹配。
- 用普通员工账号和审核人账号验证权限分离。

P2：

- 继续优化文件管理器体验。
- 增加受控演练重置入口。
- 接入 DWG 转换/查看能力。

## 11. 结论

105 启航华居项目的演练说明：平台主链路已经具备“可跑通”的雏形，但真实项目价值仍卡在“工程主数据是否真实”。如果部位树只是模板骨架，后续交付、审核、预检查都会变成形式化流程。下一阶段最重要的不是继续堆按钮，而是让 Agent 把真实目录线索、文件命名、专业、楼栋楼层线索整理成可审批的工程主数据草案，再由员工在这个真实结构下补交文件。
