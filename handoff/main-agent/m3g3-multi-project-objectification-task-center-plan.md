# M3G-3：多真实项目分批对象化策略与任务中心增强计划

更新时间：2026-05-28

## 批次定位

`M3G-3` 承接已收口的 `M3G-2：105 项目历史文件对象化上传灰度`。

M3G-2 已证明单项目小批历史文件可以复制副本到 NAS 侧 MinIO，并且对象化文件可通过受控 `file-access` 读取。M3G-3 不直接做全量迁移，而是把“多真实项目分批对象化”做成可规划、可限速、可审计、可回滚判断的任务中心能力。

## 本批目标

让平台可以安全回答：

1. 哪些真实项目适合先对象化。
2. 每个项目对象化预计文件数、容量、类型分布、风险是什么。
3. 如果按项目 / 目录 / 文件类型 / 文件大小筛选，会选中多少文件和容量。
4. 当前任务中心是否支持批量计划、限速、并发上限、暂停 / 继续 / 重试 / 跳过。
5. 是否可以在明确选定的小范围项目上创建受控对象化任务。

## 关键原则

- 不做全量迁移。
- 不迁移 NAS 根目录。
- 不自动选择所有真实项目。
- 不移动、删除、重命名、覆盖真实 NAS 原文件。
- 不读取正文。
- 不写语义索引。
- 不触发 Hermes 正文问答。
- 不暴露真实路径、bucket、object key、`storage_uri`、SQL、raw row、token、secret。

## 范围定义

### 1. 多项目对象化盘点增强

在现有全项目对象化覆盖率基础上，补齐：

- 项目维度筛选：真实 NAS 项目、active 项目、归档 / 样例 / 测试项目排除。
- 文件类型分布。
- 容量分布。
- checksum 覆盖率。
- 已对象化数量。
- NAS_ONLY 数量。
- MIGRATION_FAILED 数量。
- 超大文件数量。
- 路径不可读数量。
- 预计对象化容量。

### 2. 多项目 dry-run 计划

新增或扩展 dry-run 能力：

- 支持多个 projectId。
- 支持按目录、扩展名、大小范围、checksum 状态、对象化状态筛选。
- 支持总文件数上限。
- 支持总容量上限。
- 返回按项目分组的计划结果。
- 返回风险提示和跳过原因。

默认只读，不创建迁移任务。

### 3. 任务中心增强

在现有迁移任务中心基础上增强：

- 显示任务策略：项目范围、筛选条件、文件数上限、容量上限。
- 显示任务批次来源：手动 fileIds / dry-run 计划 / 多项目计划。
- 显示任务执行摘要：成功、跳过、失败、失败原因。
- 预留并发上限 / 速率限制字段。
- 预留暂停 / 继续语义；如本批实现成本高，可只做状态与后续扩展位，不做真正 worker 暂停。
- 重试仍必须可用。

### 4. 受控小范围执行

本批允许受控执行，但必须满足：

- 用户 / 脚本显式指定项目范围。
- 默认不超过少量项目。
- 每个任务仍有文件数和容量上限。
- 不允许“所有项目一键执行”。
- 执行结果必须可追踪、可审计、可重试。

### 5. 前端

文件服务 / 对象存储页面应能展示：

- 全项目对象化概览。
- 项目对象化风险列表。
- 多项目 dry-run 计划。
- 可创建的小批任务。
- 任务中心状态。
- 明确文案：NAS 原文件保留；对象化只是复制副本到对象存储。

## 建议接口

命名以现有代码风格为准，可复用现有接口。

可新增：

- `POST /api/data-steward/storage-objectification-plans:dry-run`
  - 多项目 dry-run。
- `POST /api/data-steward/storage-migration-tasks:from-plan`
  - 从受控计划创建任务。
- `GET /api/data-steward/storage-migration-tasks`
  - 全局任务列表。

也可以不新增接口，若现有接口可安全扩展，则优先扩展。

## 测试要求

新增专项脚本：

`scripts/dev/check-m3g3-multi-project-objectification-planning.sh`

脚本默认应只读，不应真实对象化新文件，除非明确传入执行开关。

必须验证：

1. readiness 为 `NAS_SIDE_MINIO / READY`。
2. 全项目盘点可区分真实项目、样例 / 测试 / 归档项目。
3. 多项目 dry-run 可生成计划。
4. dry-run 不创建迁移任务。
5. 计划结果按项目分组。
6. 文件数 / 容量上限生效。
7. 禁出字段扫描通过。
8. 现有 M3G-2 / M3G-1 / M3F / M3E / M3C / file-access 回归通过。

如开发 agent 实现了受控执行入口，测试必须额外验证：

- 只对明确指定项目执行。
- 不越过任务上限。
- NAS 原文件未变。
- 重复执行幂等。

## 完成定义

M3G-3 完成时，不要求公司全部真实项目完成对象化。

M3G-3 只验收：

- 多项目对象化规划能力可用。
- 任务中心能支撑后续分批执行。
- 批量策略有文件数、容量、项目范围控制。
- 无敏感字段泄露。
- 真实 NAS 原文件未被破坏。
