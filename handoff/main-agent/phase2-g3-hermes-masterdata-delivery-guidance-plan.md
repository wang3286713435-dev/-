# G3：Hermes 工程主数据交付路径完善 MVP

更新时间：2026-05-20

## 1. 主 Agent 裁决

当前不进入 `8B：BIM 引擎接入前置与轻量化任务编排骨架`。

原因：

- 继续增加轻量化、构件级、NAS 写操作等新能力，会让平台变得更大，但真实项目交付路径仍然不够顺。
- 当前更紧急的问题是：员工进入真实项目后，不理解工程主数据、交付标准、缺失项、挂接、审核之间的关系。
- Hermes 已接入真实外部 Agent，但还没有充分成为“懂平台、懂当前项目、懂工程主数据流程”的常驻引导者。

因此，G2 收口后下一阶段固定为：

`G3：Hermes 工程主数据交付路径完善 MVP`

G3 是主线可用性修复批次，不是新功能扩张批次。

## 2. G3 目标

让用户进入 105 或其他真实 NAS 项目后，即使不看说明书，也可以通过 Hermes 理解：

- 当前项目处于真实接入 / 工程主数据 / 交付治理的哪一步。
- 工程主数据为什么要配置。
- 部位树、节点类型、交付物标准分别有什么作用。
- 当前项目缺少哪些前置条件。
- 当前页面下一步应该做什么。
- 哪些问题属于平台目录级可回答，哪些问题缺少正文证据。

一句话验收：

> 一个普通员工进入真实项目后，只问 Hermes，也能知道工程主数据怎么配置、交付缺什么、下一步该点哪里。

## 3. G3 当前允许范围

G3 只允许强化已有能力：

- Hermes 常驻入口。
- 当前项目 / 当前页面上下文注入。
- 工程主数据页面的业务解释。
- 真实项目接入向导的草案语义解释。
- 交付治理助手的下一步建议。
- 文档 / 图纸交付缺失项解释。
- catalog-only 项目目录级问答。
- Missing Evidence 解释。
- 前端文案、路由上下文、快捷问题和回答展示优化。
- 只读聚合上下文，优先复用现有接口和服务。

## 4. G3 明确禁止范围

G3 不做：

- BIM 轻量化引擎。
- 真实模型转换。
- 构件级解析。
- PDF / Office / DWG / RVT / IFC 正文抽取。
- selective indexing。
- 向量库、OpenSearch、Qdrant、MinIO documents/chunks 写入。
- Hermes memory 写入文件正文、raw catalog row 或真实 NAS 路径。
- 真实 NAS 增删改查。
- Agent 自动写库。
- Agent 自动创建部位树、节点类型或交付标准。
- Agent 自动挂接、自动审批、自动整改。
- 大量新增页面。

## 5. 核心设计

### 5.1 区分两类问题

G3 必须让 Hermes 正确区分：

1. 平台使用 / 当前项目流程问题
   - 例如：这个页面是干什么的？我下一步该做什么？部位树有什么用？
   - 这些可以基于平台上下文、工程主数据状态、交付状态回答。
   - 不应因为没有文件正文而返回 Missing Evidence。

2. 文件正文 / 图纸 / 模型内部内容问题
   - 例如：这个 DWG 里有哪些设备？RVT 里有哪些构件参数？PDF 第三页写了什么？
   - 当前必须返回 Missing Evidence。
   - 不得编造正文或模型内部内容。

### 5.2 工程主数据路径解释

Hermes 必须能解释当前数字化交付路径：

`真实项目资产 -> 接入评估 -> 主数据草案 -> 部位树确认 -> 节点类型锁定 -> 交付物标准确认 -> 文档/图纸缺失项 -> 候选文件推荐 -> 人工确认挂接 -> 审核/整改 -> 导出预检查`

说明必须面向业务用户，不堆技术字段。

### 5.3 页面级上下文

至少覆盖以下页面：

- 资产总览。
- 项目工作台。
- 真实项目接入向导。
- 工程管理部位树。
- 节点类型。
- 交付物标准。
- 交付治理助手。
- 文档交付。
- 图纸交付。
- 文件管理 / 资产目录。

Hermes 面板需要知道：

- 当前路由。
- 当前页面用途。
- 当前项目 ID / 编码 / 名称。
- 当前项目是否真实 NAS 项目。
- 当前页面下一步动作。
- 当前阶段缺口。

### 5.4 只读上下文来源

优先复用已有能力：

- 项目列表 / 项目分类。
- 真实项目接入 assessment / preview。
- 工程主数据已有状态。
- G1 交付治理 overview / missing-items。
- 交付包预检查。
- catalog-only 文件目录信息。

如确实需要新增后端聚合，只能新增只读、脱敏、项目权限校验后的上下文摘要，不新增数据库结构。

## 6. 验收样本

必须至少覆盖两个真实 NAS 项目：

- 105 项目只作为样本，不允许硬编码。
- 另选至少一个真实 NAS 项目，例如 93 / 100 / 104 / 108 / 109。

## 7. P0 / P1

P0：

- Hermes 在有项目权限时仍误报权限拒绝。
- Hermes 对平台使用问题仍只返回正文 Missing Evidence。
- Hermes 编造文件正文、DWG/RVT/BIM 内容。
- 前端直连 Hermes。
- 泄露真实 NAS 路径、raw row、SQL、token。
- Agent 自动写库或自动治理。

P1：

- 工程主数据页面没有明确 Hermes 引导。
- Hermes 回答无法说明当前页面下一步动作。
- 只对 105 有效，其他真实项目不可用。
- 用户仍无法理解部位树、节点类型、交付物标准与交付闭环的关系。

## 8. 后续路线

G3 收口后，建议进入：

1. `G4：真实项目交付闭环试运行与问题修补`
2. 然后再考虑：
   - `8B：BIM 轻量化任务编排骨架`
   - `Evidence Layer：正文证据检索`

8B 当前后置，不作为下一批默认主线。
