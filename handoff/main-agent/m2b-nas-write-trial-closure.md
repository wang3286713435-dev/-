## 1. 收口结论

`M2B：受控 NAS 写操作真实项目灰度试运行与安全开关` 可以正式收口。

- 默认无灰度配置时，真实项目写操作关闭。
- 真实项目文件管理页能展示写入灰度状态、禁用原因和风险提示。

- M2A / M1F / M1E / M1D / M1C 回归均通过。

- 未发现 raw NAS 路径、`storage_uri`、`storage_path`、raw row、SQL、token、secret、password 泄露。

## 3. 真实项目边界

本轮没有把真实项目写操作默认打开。

真实项目必须显式满足以下条件后，才允许进入受控写入：

- 项目级写操作灰度开关已开启。
- 当前操作目录在允许写入范围内。
- 当前账号具备项目内可写角色或被纳入允许账号。
- 写接口继续通过后端校验，不依赖前端隐藏按钮。

当前收口仅证明灰度机制和安全提示可用，不代表公司全部真实项目已开放 NAS 写操作。

## 4. 继续禁止事项

M2B 收口后仍禁止：

- 永久物理删除。
- 跨项目移动。
- 批量删除 / 批量移动 / 批量重命名。
- Agent / Hermes 触发 NAS 写操作。
- 前端传入或展示真实 NAS 绝对路径。
- 文件正文读取、parser、indexing、OpenSearch / Qdrant / MinIO documents/chunks 写入。
- Hermes / G4 / 8B / 8C / 9A 扩展。

## 5. 主 Agent 裁决

M2B 可以收口，主线健康度维持 `绿灯`。

下一步不建议继续扩展 NAS 写能力。按既定顺序，应先做一次 Git checkpoint / push，然后进入：

`UX1：前端路由逻辑与视觉体验专项优化批次`

UX1 仍需保持既定边界：只调整前端路由、菜单、项目工作台、组件布局、样式、交互文案和页面体验；不得修改后端业务逻辑、接口语义、数据库迁移或权限规则。
