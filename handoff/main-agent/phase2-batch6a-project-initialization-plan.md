# 二期批次 6A：客户项目初始化与标准模板化最小闭环

时间：2026-05-18

## 1. 主 Agent 裁决

`5B 数据管家客户版模块补齐` 已正式收口，下一步进入 `批次 6：客户项目初始化与标准模板化`。

本轮不一次性做完整客户初始化平台，而先拆出 `6A`：

`项目初始化评估 + 建筑机电/BIM 交付标准模板预览 + 受控套用 + 标准就绪导航`

目标是让客户或项目管理员进入一个已有项目后，可以清楚看到：

- 当前项目离“标准就绪”还差什么。
- 平台内置的建筑机电/BIM 模板包含什么。
- 模板套用后会创建哪些部位、节点类型、交付物标准和目录模板。
- 套用过程不覆盖、不删除已有客户数据。
- 套用完成后能继续锁定节点类型、维护交付标准，并进入文档/图纸交付。

## 2. 为什么先做 6A

当前平台已经具备：

- 工程管理部位树 CRUD。
- 节点类型 CRUD 和锁定。
- 交付物定义、类型、属性 CRUD。
- 目录模板 CRUD。
- 标准状态接口 `standard-status`。
- 项目工作台导航。

但客户侧体验仍然有断点：

- 新项目进入后，不知道标准配置从哪里开始。
- 现有标准页面是分散维护页，不像“初始化向导”。
- 目录模板目前只能手工录 JSON，客户不容易理解。
- 没有“一键套用建筑机电基础模板”的入口。

因此 6A 要把已有能力组织成客户可执行流程，而不是新增重型业务。

## 3. 6A 范围

### 后端

优先复用现有表和服务，不新增迁移；如确实必须新增表，只能追加 `V21`，不得修改旧迁移。

建议新增 `master-data` 初始化应用服务和接口：

- `GET /api/master-data/projects/{projectId}/initialization/status`
- `GET /api/master-data/standard-templates`
- `GET /api/master-data/standard-templates/{templateCode}`
- `POST /api/master-data/projects/{projectId}/initialization:preview-template`
- `POST /api/master-data/projects/{projectId}/initialization:apply-template`

内置模板先用代码或资源文件维护，不强制上数据库：

- 模板编码：`MEP_BIM_BASIC`
- 模板名称：`建筑机电 BIM 交付基础模板`
- 适用范围：建筑机电/BIM交付

模板内容至少覆盖：

- 部位树：项目 / 单体 / 楼层 / 系统 或等价的建筑机电常用结构。
- 节点类型：项目、单体、楼层、系统、设备或等价节点。
- 交付物定义：模型、图纸、文档、资料包等基础交付项。
- 交付物类型：RVT/IFC/NWD/NWC、DWG/PDF、Office/PDF 等。
- 交付物属性：专业、版本、阶段、责任人、审核状态等基础字段。
- 目录模板：文档交付目录、图纸交付目录、模型目录基础 JSON。

套用规则：

- 幂等：按 code/name 判断已存在内容，不重复创建。
- 非破坏：不得删除、覆盖、改名已有客户数据。
- 不越权：必须校验当前项目上下文。
- 不跳锁：如果需要创建交付物定义/目录模板，必须按现有规则先锁定节点类型；可以由接口在模板套用流程中创建节点类型后自动锁定，也可以明确分成两步，但前端必须解释清楚。
- 审计：预览不审计；套用必须写审计，包含模板编码、创建数量、跳过数量、操作人、项目。

### 前端

在项目工作台内新增项目初始化入口，建议路由：

- `/data-steward/assets/:projectId/master-data/initialization`

入口位置：

- 项目工作台顶部 `工程主数据` 分组新增 `初始化向导`。
- 工程主数据页面中可以提供返回/继续入口。

页面结构：

- 顶部：当前项目、标准就绪状态、下一步建议。
- 步骤卡：
  1. 项目基础状态。
  2. 部位树。
  3. 节点类型。
  4. 交付物标准。
  5. 目录模板。
  6. 锁定与进入交付。
- 模板区：
  - 模板列表。
  - 模板预览。
  - 预计新增/跳过数量。
  - 套用按钮。
- 套用结果：
  - 创建了什么。
  - 跳过了什么。
  - 下一步去哪里。

## 4. 6A 禁止事项

本批严禁：

- 不做真实 NAS 上传、新建文件夹、移动、重命名、删除。
- 不做 Office/CAD/BIM 文件转换。
- 不做 BIM 轻量化。
- 不做构件级解析、构件级搜索。
- 不做正文抽取、向量库、selective indexing。
- 不做 Agent 自动初始化、自动审批、自动写库。
- 不做复杂多模板市场。
- 不做跨行业模板，只做建筑机电/BIM交付基础模板。
- 不重做全部工程主数据页面。

## 5. 验收链路

最小验收链路：

`进入项目 -> 打开初始化向导 -> 查看当前标准状态 -> 查看 MEP_BIM_BASIC 模板 -> 预览将创建/跳过内容 -> 套用模板 -> 标准状态刷新 -> 节点类型锁定 -> 交付物标准和目录模板可见 -> 文档/图纸交付页面能继续使用`

P0：

- 套用模板覆盖或删除已有客户数据。
- 无权限项目可被初始化。
- 跳过项目上下文校验。
- 创建重复编码导致后续标准页面不可用。
- 套用后 `standard-status` 与真实数据不一致。
- 破坏已收口的 4R/5A/5B。

P1：

- 页面没有清楚告诉用户当前缺什么。
- 模板预览不展示将创建/跳过的内容。
- 套用结果不可追踪。
- OpenAPI 缺少新增接口。
- 审计缺失。

3. 前端初始化向导页面和项目工作台入口。
