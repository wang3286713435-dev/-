# UX2：前端使用逻辑与体验重构专项计划

更新时间：2026-05-22

## 1. 批次定位

`UX2` 是 UX1 之后的前端专项批次，目标不是继续单纯美化 UI，而是重构用户进入平台后的使用逻辑。

UX2 启动条件：

- UX1 测试 agent 报告已写入 `handoff/test-agent/latest-report.md`。
- UX1 无 P0 / P1。
- 若 UX1 存在 P0 / P1，必须先由 Claude Code 修复 UX1 阻塞项，再进入 UX2。

UX2 仍然只允许修改：

- `frontend/**`
- `handoff/dev-agent/latest-report.md`

UX2 禁止：

- 修改 `backend/**`。
- 修改数据库迁移。
- 新增或修改后端接口。
- 改变权限规则。
- 修改 `docs/**`。
- 新增 Hermes / BIM / NAS / parser / indexing / 文件正文读取能力。

## 2. 核心问题

UX1 已改善视觉 token、壳层样式和三段导航，但平台仍可能存在：

- 用户进入平台后不知道先看什么。
- 资产总览统计、风险、项目列表和动作入口信息量过大。
- 项目工作台同时出现太多入口，主次关系不够强。
- 文件管理、工程主数据、交付工作中心仍暴露过多技术字段。
- 一些低频入口抢占主视觉，常用入口不够醒目。
- 视觉还偏稳重，缺少让人愿意使用的现代质感。

## 3. 产品目标

让普通员工 10 秒内理解：

1. 先选哪个真实项目。
2. 当前项目下一步该做什么。
3. 哪些资料已经可用。
4. 哪些主数据还没确认。
5. 哪些交付项需要补交、审核或整改。

最终主路径固定为：

`选择项目 -> 看资产 -> 确认主数据 -> 做交付 -> 查整改 / 预检查`

## 4. UX2 分批

### UX2.0：UX1 验收处理

- 如 UX1 测试报告存在 P0 / P1，先修复。
- 清理或明确忽略 `.claude/**`、`CLAUDE.md` 等非交付文件。
- 不允许把 `.claude/**`、`CLAUDE.md` 默认纳入 UX2 提交。

### UX2.1：入口与资产总览重构

- 资产总览改成真正的项目入口台。
- 主视觉只保留：
  - 真实项目。
  - 待接入项目。
  - 需要处理的项目。
  - 最近进入项目。
  - 下一步动作。
- 弱化或折叠统计堆叠、风险明细和低频筛选。

### UX2.2：项目工作台重构

- 默认展示“当前项目最重要状态 + 下一步动作”。
- 项目资产、工程主数据、交付工作中心不再平铺所有入口。
- 常用功能前置，低频功能进入“更多工具”。
- 每个关键页面都提供“返回项目工作台”和“下一步推荐动作”。

### UX2.3：关键页面减负

- 文件管理默认展示：文件名、类型、版本、大小、专业、状态、操作。
- checksum、扫描任务编号、诊断编号、内部 ID 等技术字段进入“技术信息 / 诊断信息”。
- 初始化向导、文档交付、图纸交付保留业务链路，减少技术词和重复说明。
- 不删除数据能力，只做默认隐藏、折叠、移入详情或高级区域。

### UX2.4：现代视觉质感提升

- 只在壳层、资产总览 Hero、项目状态卡、下一步行动卡引入轻量 glass-lite。
- 表格、权限、NAS 写操作、交付审核保持清晰企业中后台风格。
- 加入柔和边框、浅阴影、层叠卡片、现代留白和更清晰的视觉焦点。
- 禁止大面积玻璃化、强模糊、炫光、霓虹、营销页风格。

## 5. 验收标准

UX2 收口必须满足：

- 前端构建通过。
- 后端健康检查通过。
- `git diff --check` 通过。
- M2B / M2A / M1F / M1E / M1D / M1C 回归通过。
- Fresh login 进入资产总览，用户能在 10 秒内理解下一步。
- 503 / 506 项目工作台能清楚看到“当前项目状态”和“下一步动作”。
- 文件管理默认不被技术字段、后台任务、扫描信息干扰。
- 文档 / 图纸交付能看懂应交、缺失、补交、审核、整改、预检查关系。
- 旧链接兼容跳转仍可用。
- 1280 / 1440 / 1920 多分辨率无横向溢出、按钮丢失、文字挤压。

## 6. P0 边界

出现以下任一情况，UX2 不得收口：

- 修改后端、数据库、docs 或接口语义。
- 删除旧路由导致白屏。
- 项目上下文错乱或串项目。
- 真实 NAS 路径泄露。
- M2B / M2A / M1F / M1E / M1D / M1C 回归失败。
- 为 503 / 506 / 105 / 93 写死逻辑。
