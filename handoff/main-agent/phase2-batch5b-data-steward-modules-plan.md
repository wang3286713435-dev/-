# 二期批次 5B：数据管家客户版模块补齐计划

时间：2026-05-18

## 1. 目标

5B 的目标是把当前“资产驾驶舱 + 文件管理 + 贾维斯入口”补成客户能理解的数据管家完整工作区。

本批不追求新增复杂底层能力，而是把已有后端能力、一期数据治理能力和二期前端基线组织成清晰的项目内模块：

- 资产驾驶舱
- 文件管理
- 模型集成
- 管理对象
- 事项列表
- 任务列表
- 导出列表
- 文件服务

用户进入某个项目后，应能在数据管家里看到“项目资产在哪里、模型如何集成、对象如何管理、治理事项和任务如何追踪、导出和文件服务当前可用到什么程度”。

## 2. 当前基础

已具备：

- 项目工作台与项目内导航。
- 数据管家资产驾驶舱。
- 文件管理：左目录树、右文件表、高级搜索、文件详情、预览、治理、补 checksum。
- 4R 文件访问安全：预览/下载权限分离、路径隐藏、短时票据、审计。
- 5A / 5A.1 贾维斯数据管家：只读问答、Missing Evidence、health、平台语义 chat。
- 后端已有模型集成与管理对象基础 API：
  - `/api/data-steward/projects/{projectId}/model-integrations`
  - `/api/data-steward/projects/{projectId}/managed-objects`
- 后端已有任务、扫描任务、事件流、文件访问和统计接口，可支撑任务列表、事项列表、导出列表和文件服务的只读/占位能力。

当前缺口：

- 模型集成和管理对象页面仍像早期 MVP，未融入项目级数据管家体验。
- 事项列表、任务列表、导出列表、文件服务未形成客户可见模块。
- 数据管家侧栏/页签没有完整呈现 RealBIM 风格的信息架构。
- 一些入口仍偏“技术接口清单”，客户不容易理解模块价值。

## 3. 本批范围

### 3.1 模型集成

目标：做成客户可理解的模型资产汇聚页面。

要求：

- 在项目数据管家内部可进入。
- 展示集成名称、关联模型文件、版本、状态、构件数、发布时间。
- 可从当前项目模型文件创建模型集成。
- 可发布集成，但只改变平台元数据状态，不做真实 BIM 轻量化。
- 页面说明清楚：当前为模型资产登记/集成管理，不是在线三维预览。
- 保留后续 BIM 引擎适配入口，但禁用真实轻量化动作。

### 3.2 管理对象

目标：做成客户可理解的对象台账页面。

要求：

- 在项目数据管家内部可进入。
- 展示对象编码、对象名称、类型、专业、关联模型、关联部位、状态。
- 可创建、编辑、停用/删除平台对象记录。
- 不做构件级解析，不从模型自动抽构件。
- 支持从已发布模型集成中选择模型。

### 3.3 事项列表

目标：呈现数据管家的治理事项和待处理问题。

要求：

- 以只读/半只读方式聚合现有数据：
  - 缺 checksum
  - 缺专业
  - 低置信度
  - 待审核候选
  - 扫描失败
  - 路径失效
  - 贾维斯 Missing Evidence 可作为后续扩展，不在本批强制接入
- 每条事项给出类型、项目、对象/文件、严重程度、状态、建议动作。
- 可跳转到对应文件详情、扫描任务或治理入口。
- 不做自动修复，不做 Agent 自动处理。

### 3.4 任务列表

目标：把后台任务变成客户能看懂的任务中心。

要求：

- 复用 `assets/jobs` 和扫描任务接口。
- 展示任务类型、状态、进度、失败原因、创建时间、更新时间。
- 支持筛选任务类型和状态。
- 支持失败任务重试，但必须沿用后端已有权限和审计。
- 不新增自动调度能力。

### 3.5 导出列表

目标：提供客户可理解的导出入口和导出历史外壳。

要求：

- 本批优先实现前端模块和最小导出能力。
- 可复用已有 CSV 报表或文件列表导出能力。
- 如果后端没有导出任务表，不新增复杂表结构；先做“即时导出 + 本页操作记录/最近事件提示”。
- 导出内容不包含真实 NAS 绝对路径，普通用户不得看到底层路径。
- 不做异步大批量打包下载。

### 3.6 文件服务

目标：把平台当前可提供的文件服务能力汇总展示。

要求：

- 展示预览、下载、权限证明、路径隐藏、访问票据、文件详情、受控访问的能力状态。
- 提供入口跳转到文件管理、权限证明、贾维斯问答。
- 明确禁用或说明未开放能力：
  - 上传
  - 新建文件夹
  - 移动
  - 重命名
  - 真实删除
  - 批量打包下载
- 不开放真实 NAS 写操作。

## 4. 路由与导航

优先在项目工作台内组织，不恢复全局割裂导航。

建议项目内路径：

- `/data-steward/assets/:projectId/data-steward/dashboard`
- `/data-steward/assets/:projectId/data-steward/files`
- `/data-steward/assets/:projectId/data-steward/models`
- `/data-steward/assets/:projectId/data-steward/objects`
- `/data-steward/assets/:projectId/data-steward/issues`
- `/data-steward/assets/:projectId/data-steward/tasks`
- `/data-steward/assets/:projectId/data-steward/exports`
- `/data-steward/assets/:projectId/data-steward/file-service`

可兼容旧路由：

- `/data-steward/models`
- `/data-steward/objects`

但旧路由不应继续成为主入口；如进入，应尽量基于当前项目跳转到项目工作台内路由。

## 5. 禁止事项

本批严禁：

- 真实 NAS 写入、移动、重命名、删除、新建文件夹。
- 模型轻量化、构件级解析、构件级搜索。
- PDF / Office / CAD / BIM 正文抽取。
- Agent 自动治理、自动审批、自动修复。
- 写 OpenSearch / Qdrant / MinIO。
- 大改后端核心模型或一期已收口数据治理逻辑。
- 为了页面好看绕过权限、路径隐藏或审计。

## 6. 验收标准

1. 项目工作台内能看到完整数据管家模块入口。
2. 资产驾驶舱、文件管理、模型集成、管理对象、事项列表、任务列表、导出列表、文件服务都能打开。
3. 模型集成和管理对象使用当前项目上下文，不依赖全局项目切换。
4. 事项列表能反映当前项目的数据质量/治理问题，并能跳转到对应处理入口。
5. 任务列表能展示扫描、checksum 等后台任务状态、进度、失败原因和重试入口。
6. 导出入口不泄露真实 NAS 路径。
7. 文件服务页清楚说明当前已开放能力和未开放写操作。
8. 页面不横向撑爆，大项目不全量卡死。
9. 4R 文件访问安全、5A/5A.1 贾维斯只读边界不回归。
10. 前后端构建通过，专项脚本通过。

## 7. 测试建议

应新增或增强一个 5B 专项脚本，例如：

`scripts/dev/check-phase2-batch5b-data-steward-modules.sh`

覆盖：

- 登录并切换项目。
- 模型集成列表/创建/发布。
- 管理对象列表/创建/编辑。
- 任务列表查询和失败任务重试权限。
- 事件/质量数据可支撑事项列表。
- 导出接口或即时导出入口不含真实 NAS 路径。
- OpenAPI 包含关键 API。
- 4R 和 5A.1 脚本回归。

前端需要人工或浏览器回归：

- 项目工作台进入 8 个数据管家模块。
- 模块间切换不丢项目上下文。
- 页面无横向溢出。
- 禁用的 NAS 写操作不会误导为已开放。
