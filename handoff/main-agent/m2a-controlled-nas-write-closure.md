# M2A NAS 受控文件操作安全底座收口报告

## 1. 收口结论

`M2A：NAS 受控文件操作安全底座` 已通过测试 agent 复核，可以正式收口。

当前无 P0 / P1 / P2。

本轮收口范围为：

`低风险写操作 + 删除到隔离区，不开放永久删除与批量操作`

## 2. 已验证能力

已通过专项脚本和回归验证：

- 项目内新建文件夹。
- 项目内上传文件。
- 项目内文件 / 文件夹重命名。
- 项目内文件 / 文件夹移动。
- 删除到隔离区。
- 隔离区恢复。
- 操作记录。
- 审计日志。
- 操作后元数据同步。
- 查看者写操作拒绝。
- 未授权用户写操作拒绝。
- 路径穿越拒绝。
- forbidden 字段扫描。

`scripts/dev/check-m2a-controlled-nas-write.sh` 结果：

`PASS=20 FAIL=0`

## 3. 回归结果

已通过：

- 后端构建。
- 前端构建，仅保留既有 Vite chunk size warning。
- 后端健康检查。
- M1F 员工注册与权限管理回归。
- M1E 文件管理连续工作体验回归。
- M1D 标准驱动交付闭环回归。
- M1C 工程主数据真实项目落地回归。
- Phase2 batch4 文件访问安全回归。
- `git diff --check`。

## 4. 真实 NAS 风险确认

测试 agent 已确认：

- M2A 专项脚本只在脚本创建的 `/tmp/delivery-m2a-nas-*` 隔离根目录内执行写操作。
- 真实项目页面只做只读人工检查。
- 未在真实业务 NAS 目录执行上传、移动、删除、重命名等写操作。
- 真实项目 503 的 `隔离区` 和 `操作记录` 抽屉可打开。
- 页面和接口响应未发现 raw NAS 路径、`storage_uri`、`storage_path`、SQL、token、secret、password 泄露。

## 5. 禁止边界仍然有效

M2A 收口后仍禁止：

- 永久物理删除。
- 跨项目移动。
- 批量删除 / 批量移动 / 批量重命名。
- Agent / Hermes 触发 NAS 写操作。
- 前端传入或展示真实 NAS 绝对路径。
- 文件正文读取。
- 真实 BIM 轻量化、构件解析。
- G4 / Hermes 新能力 / 8B / 8C / 9A。

## 6. 主 agent 裁决

M2A 可以正式收口。

主线健康度保持：

`绿灯可继续主线开发`

但 M2A 只是安全底座通过，不代表已经可以面向所有员工、所有真实项目开放完整 NAS CRUD。

下一步建议进入：

`M2B：受控 NAS 写操作真实项目灰度试运行与安全开关`

M2B 应先选择一个低风险真实项目或项目内试运行目录，明确可操作目录、可操作账号、备份 / 恢复方式和操作清单，再逐步让员工试用。
