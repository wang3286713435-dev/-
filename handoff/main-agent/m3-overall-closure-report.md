## 3. M3 已完成能力

### 3.1 存储底座

- 建立对象存储数据模型。
- 建立 `StorageService` 适配层。
- 支持 NAS 与 NAS 侧 MinIO 的受控读取。
- 支持对象存储健康检查。
- 已对象化文件有 active object version。

### 3.2 新文件写入

- 新上传文件默认优先写入对象存储。
- 对象存储不可用时 fail-closed，不静默伪成功。
- 文件台账、对象台账、active 对象版本同步写入。

### 3.3 历史文件对象化

- 支持 dry-run 规划。
- 支持受控小批执行。
- 支持 105 长跑、暂停、继续、重试。
- 支持全项目覆盖率报告。
- 不移动、不删除、不重命名、不覆盖真实 NAS 原文件。

### 3.4 读取链路

- 已对象化文件默认走 NAS 侧 MinIO。
- NAS_ONLY 文件仍可通过历史 NAS 链路读取，但状态必须明确。
- 对象副本不可读时不静默 fallback 到 NAS 冒充成功。
- `file-access` 仍是前端访问文件的唯一受控入口。

### 3.5 安全边界

- 前端和普通 API 不返回真实 NAS 路径。
- 不返回 `storage_uri`。
- 不返回 bucket / object key。
- 不返回 raw row、SQL、token、secret。
- 高风险动作保留权限校验和审计边界。
