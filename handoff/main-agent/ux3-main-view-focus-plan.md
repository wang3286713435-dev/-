# UX3 主视图聚焦与认知减负计划

## 背景

UX2 已完成并通过整体验收，但用户反馈当前主视图仍然过于复杂：教程、说明文字和功能入口过多，员工进入后不知道先看什么、先点什么。

本批次属于 UX 分支内的小批次，不进入 M2C，不新增平台业务能力。

## 目标

把平台主视图改成更直接的工作入口：

- 资产总览是“项目启动台”，首屏只服务于选项目和进入核心功能。
- 项目工作台首屏聚焦 `文件管理`、`项目资产驾驶舱 / 可视化`、`交付状态`。
- 低频功能保留，但不抢主视觉。
- 长教程和技术说明从主视图移到折叠区、详情区或空状态中。

## 当前批次

- 批次名：`UX3：主视图聚焦与认知减负`
- 分支：`codex/ux3-main-view-focus`
- 基线：UX2 收口提交 `b5cfb01`

## 边界

允许修改：

- `frontend/**`
- `handoff/dev-agent/latest-report.md`

禁止修改：

- `backend/**`
- 数据库迁移
- 后端接口语义
- 权限规则
- `docs/**`
- Hermes / BIM / NAS 写能力

## 设计原则

- 做减法，不做新功能。
- 用交互和入口层级引导用户，少用教程文字。
- 核心功能集中在首屏：项目管理、文件管理、项目可视化。
- 旧路由继续兼容，不删除入口，只降低低频入口权重。
- 保留 UX2 的视觉升级，不回滚 lightfield / spotlight / glass-lite。
