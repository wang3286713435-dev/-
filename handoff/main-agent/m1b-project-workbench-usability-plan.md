# M1B 项目工作台与数据管家可用性收口计划

时间：2026-05-20

## 批次定位

`M1B：项目工作台与数据管家可用性收口`

M1B 是 M1A 之后的主线平台可用性批次。目标不是继续扩功能，而是让普通员工进入平台后能看懂：

- 哪些项目是真实 NAS 项目。
- 当前项目处于什么接入 / 主数据 / 交付状态。
- 每个功能入口是做什么的。
- 下一步应该点哪里。

## 背景问题

M1A 已经证明主线基础链路可用，但还留下一个核心体验问题：

平台页面能力很多，但用户仍容易感觉“功能铺开了，不知道哪个有什么用”。

测试 agent 也记录了一个非阻塞观察项：

- `/data-steward/assets` 默认真实项目统计显示为 0。

这类问题不一定是 P0/P1，但会直接影响真实员工试用。

## 本批目标

1. 优化资产总览 Hero 区，让用户一进系统能理解当前项目资产状态。
2. 优化项目卡片 / 列表的信息层级，清楚展示项目来源、接入状态、主数据状态、交付状态和下一步动作。
3. 优化项目工作台顶部区域，让用户知道“我在当前项目里工作”以及“数据管家 / 工程主数据 / 工作中心分别做什么”。
4. 修复真实项目统计显示明显不准的问题。
5. 改善空状态、提示文案和按钮文案，减少技术字段直出。

## 允许范围

- 前端可用性调整。
- 小范围只读聚合或统计口径修复。
- 资产总览 / 项目工作台 / 文件管理 / 项目内导航文案优化。
- 真实项目状态展示修复。
- 不影响底层数据模型的小型接口修复。

## 禁止范围

- 不新增大模块。
- 不新增 Hermes 能力。
- 不进入 G4。
- 不进入 8B / 8C / 9A。
- 不做真实 NAS 增删改查。
- 不读取文件正文。
- 不做真实 BIM 轻量化。
- 不做 Agent 自动治理。
- 不修改 `docs/**`。

## 重点页面

- `/data-steward/assets`
- `/data-steward/assets/:projectId`
- 项目工作台内的文件管理页签。
- 项目工作台顶部导航。
- 工程主数据入口说明。
- 工作中心入口说明。

## 验收重点

- Fresh login 后进入资产总览，用户能理解当前页面用途。
- 真实 NAS 项目统计不再明显错误。
- 105 / 503 与另一个真实 NAS 项目都能看出接入状态和下一步动作。
- 项目工作台顶部能解释当前项目能力分区。
- 文件管理、工程主数据、文档交付、图纸交付入口不丢失。
- 页面不依赖 Hermes 也能指导基础操作。
- 不泄露真实 NAS 路径。

## 主 Agent 裁决

M1B 可以启动。开发 agent 只做可用性收口和小范围统计修复，不允许扩大为新业务模块。
