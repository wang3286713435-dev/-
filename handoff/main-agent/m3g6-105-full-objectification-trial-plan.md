# M3G-6：105 项目全量对象化试运行计划

启动日期：2026-05-29

## 定位

`M3G-6` 的正确定位是：

> 以 105 / `projectId=503` 作为第一个完整样板项目，尽量把该项目已登记文件全部对象化，让 105 从 `NAS_ONLY` 混合状态推进到“对象存储主链路可验证”的样板项目。

上一版“受控扩大真实项目对象化覆盖率”口径过于保守，偏离了用户确认的 M3G-6 目标。该口径不能作为 M3G-6 收口标准。

## 本批目标

1. 对 105 / `projectId=503` 生成全量对象化计划。
2. 覆盖该项目全部已登记 `NAS_ONLY` 文件，而不是只挑选 5-15 个样本。
3. 按批次执行，不一次性硬冲。
4. 支持暂停、继续、重试、失败原因。
5. 已对象化文件从 NAS 侧 MinIO 读取。
6. NAS 原文件不移动、不删除、不改名。
7. 输出 105 对象化覆盖率报告：
   - 总文件数
   - 已对象化数量
   - 未对象化数量
   - 失败数量
   - 跳过数量
   - 对象化容量
   - checksum 覆盖率
   - 失败原因分类
8. 失败文件进入治理清单，不阻塞整个项目继续推进。

## 分批策略

M3G-6 不是一次请求搬完 2928 个文件，而是平台支持“105 全量计划 + 分批执行”：

- 单个批次有可配置上限。
- 支持按文件数量、容量、扩展名、目录分批。
- 每批执行后更新覆盖率。
- 可继续执行下一批，直到 105 全量完成或只剩失败 / 待治理文件。
- 重复执行必须幂等，不重复生成 active object version。

## 边界

本批禁止：

- 迁移 NAS 根目录。
- 迁移其他项目作为 M3G-6 完成条件。
- 移动、删除、重命名、覆盖真实 NAS 原文件。
- 读取 PDF / Office / DWG / RVT / IFC 正文。
- 写 documents / chunks / Qdrant / OpenSearch / Hermes memory。
- 新增 Hermes 正文问答。
- 接入或修改 BIM / 葛兰岱尔分支。
- 修改 `docs/**`。

## 对上一轮错误口径的处理

当前开发 agent 已按错误 prompt 完成了“多真实项目 15 文件受控扩大”能力。该工作可作为后续 M3G-7 或 M3G 支线能力参考，但不能算 M3G-6 完成。

主 agent 裁决：

- 不按上一轮报告收口 M3G-6。
- 让开发 agent 在当前基础上继续纠偏，实现 105 全量对象化试运行。
- 如果当前代码中的硬上限阻碍 105 分批推进，需要改成“单批上限 + 全量计划进度”，而不是固定总量 15 个文件。

## 验收

M3G-6 通过标准：

- 105 项目能生成全量对象化计划。
- 105 项目能按批次持续执行对象化。
- 已对象化文件通过 NAS 侧 MinIO + 受控 `file-access` 读取。
- NAS 原文件未移动、未删除、未改名。
- 失败文件有失败原因和治理清单。
- 105 最终状态接近全量对象化；如剩余未完成，必须是明确失败 / 不可读 / 待治理原因，而不是批次能力不足。
