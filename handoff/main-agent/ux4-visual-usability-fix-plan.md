# UX4：平台视觉与员工使用效率修复计划

时间：2026-06-04

## 1. 批次定位

`UX4` 是平台视觉与可用性修复批次。

本批只修现有前端体验，不新增业务能力。

目标不是把平台变成更大的壳子，而是让员工使用已经完成的核心功能时更清楚、更顺手。

## 2. 启动原因

当前平台核心能力已经很多：

- 项目资产。
- 文件管理。
- 对象存储与 file-access。
- 工程树与文件归属。
- 文档/图纸交付。
- BIM 协同和葛兰岱尔 Viewer。

但用户反馈当前仍存在：

- 视觉重点不够清楚。
- 员工进入平台后不知道先做什么。
- 一些页面仍有过多技术字段。
- 文件管理器和工作台仍有使用性小问题。
- 核心功能不够聚焦。

因此在继续 M3-CLOSE / M3X / M4 之前，先插入 UX4 修复。

## 3. 本批重点

### 项目入口

- 让用户第一眼知道先选项目。
- 突出真实项目、最近项目、搜索和进入项目。
- 弱化低频统计和教程式文案。

### 项目工作台

- 聚焦文件管理、项目可视化、交付状态。
- 工程主数据和对象存储进入二级区域，不抢主视觉。
- 当前项目、下一步动作、风险提示要清楚。

### 文件管理器

- 区分目录浏览和全项目搜索。
- 让对象存储状态变成业务语言。
- 右键菜单、工具栏和行内操作减少重复。
- 保持 Windows / macOS 文件管理器使用习惯。

### 工程树与交付

- 文件归属、工程树、缺失项、补交和审核之间要有清楚关系。
- 正式交付资料和过程资料/归档资料要分清楚。

### 文件服务与对象存储

- 让 105 100% 对象化和其他项目未完成对象化都能被普通员工理解。
- 不把对象化说成 AI 已理解文件。

### BIM 协同

- 默认综合驾驶舱。
- READY Viewer 状态和打开入口清楚。
- 不暗示构件级能力已经完成。

## 4. 边界

允许：

- 修改 `frontend/**`。
- 更新 `handoff/dev-agent/latest-report.md`。

禁止：

- 修改 `backend/**`。
- 修改数据库迁移。
- 修改接口语义或权限。
- 修改 `docs/**`。
- 执行对象化迁移。
- 新增 Hermes、BIM、parser、indexing 能力。

## 5. 验收标准

- Fresh login 进入资产总览后，员工能在 5 秒内知道先选项目。
- 105 文件管理器目录浏览、搜索、预览、Viewer 入口不回归。
- 文件服务对象化状态更容易理解。
- 文档/图纸交付关系更清楚。
- BIM 协同综合驾驶舱和 READY Viewer 不回归。
- 旧链接兼容。
- 无横向溢出和主要按钮丢失。
- 前端构建、健康检查、M3G-9、M3G-8、file-access 回归通过。

## 6. 后续关系

UX4 完成后再回到：

1. `M3-CLOSE`：M3 整体收口判断。
2. `M3X`：全项目对象化独立批次。
3. `M4`：语义证据层。

UX4 不替代 M3-CLOSE，也不代表客户版交付准备完成。

## 7. UX4 后功能待办

UX4 合并回主线后，再评估启动独立功能批次：

- `PLM-1：项目生命周期管理 MVP`
  - 添加项目。
  - 删除/归档项目。
  - 删除需要二次确认。
  - 仅超级管理员可删除。
  - 添加项目需创建对象存储工作区和工程树根目录。
  - 详细待办见：`handoff/main-agent/post-ux4-project-lifecycle-todo.md`。

该待办不进入 UX4 当前实现范围。
