# 主 Agent 裁决记录

## 2026-05-06：长期会话协作模式

裁决：

- 主 agent 不再创建子 agent 执行开发或测试任务。
- 开发 agent 和测试 agent 必须作为长期独立会话运行。
- 跨会话交接以 `handoff/` 目录文件为准。

原因：

- 临时子 agent 完成任务后上下文关闭，不适合承载长期项目记忆。

## 2026-05-06：项目上下文规则

裁决：

- 保留“路径项目必须等于 token 当前项目”的规则。
- 项目级业务接口默认只允许访问 token 中的当前项目。
- 未来如果需要跨项目查询，必须设计明确的管理类接口，不复用项目上下文接口。

原因：

- 私有化交付平台的项目隔离是核心安全边界。
- 当前竞品也强依赖项目上下文，前端项目切换是用户操作的主入口。
- 早期收紧规则能减少后续资源、交付绑定、审计日志串项目的风险。

## 2026-05-06：第二阶段主 Agent 复查裁决

裁决：

- 节点类型锁定后不做显式解锁接口。
- 后续如果标准需要调整，走“新版本标准”机制，而不是修改已锁定记录。
- 部位树删除/停用保留当前“自身及下级级联停用”的策略。
- 菜单权限本轮够用，按钮级权限后续再细化。
- 第二阶段验收前必须修复 `masterdata_section_nodes` 逻辑删除唯一键冲突风险。

发现：

- 当前唯一键 `(project_id, code, deleted)` 会在“创建编码 A -> 删除 -> 再创建编码 A -> 再删除”时产生第二条 `deleted=1` 记录，存在唯一键冲突风险。

要求：

- 开发 agent 小范围修复该风险。
- 补充脚本验证同编码二次创建/删除场景。
- 修复完成后再交给测试 agent 验收。

## 2026-05-06：第二阶段下一批范围

裁决：

- 下一批开发范围锁定为交付物标准与目录模板最小闭环：
  - `DeliverableDefinition`
  - `DeliverableType`
  - `DeliverableAttribute`
  - `DirectoryTemplate`
- 本批不进入数据管家、文件上传、模型集成、管理对象、交付视图或工作中心正式业务。
- 交付物标准写操作必须要求项目节点类型已全部锁定；查询与状态接口可用于展示前置条件缺失。
- 目录模板本批只做项目级文档/图纸模板的结构化配置，不做真实文件目录生成。

原因：

- 竞品主链路是“节点类型锁定 -> 交付物定义/类型/属性 -> 文件/模型上传”，当前应先补齐标准底座，再放开数据管家。

## 2026-05-07：三期路线与企业 Agent 对接裁决

裁决：

- 项目正式拆为三期：一期内部 BIM 资产管理试点，二期客户交付完整版，三期增值服务和持续演进。
- 一期优先级调整为：几百个 BIM 项目建档、NAS 原地接管、元数据检索、企业 agent 可读取稳定检索底座。
- 二期必须覆盖标准驱动交付、文件预览、模型轻量化、构件级解析/搜索、审核流、项目驾驶舱、私有化部署和客户交付文档。
- 企业 agent 一期通过稳定读模型或只读宽表读取数据库；平台同时保留 REST/OpenAPI，避免 agent 直接耦合易变业务底表。
- BIM 引擎策略采用适配层先行，不在当前文档中绑定具体厂商。

原因：

- 公司内部先要解决几百个项目和约 10TB 模型资产“找得到、管得住、可检索”的问题。
- 企业 agent 仍在 MVP 阶段，需要平台提前提供稳定字段和同步口径，避免两边后续对接困难。
- 客户交付版不能按内部试点标准验收，必须单独定义完整功能、缺陷门槛和交付文档。
