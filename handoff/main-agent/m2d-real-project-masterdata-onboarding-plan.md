# M2D：真实项目工程主数据接入草案增强计划

## 背景

105 项目此前通过初始化向导套用了内置模板，形成了演示性质的工程主数据和交付标准。用户确认这会误导员工认为真实项目已完成数字化交付配置。主 agent 已将 105 项目演示模板数据软删除，文件资产仍保留。

当前 105 项目现状：

- 项目：`503 / 105 / 深圳市二十八高项目`
- 文件资产：2928 条
- 主要文件：
  - DWG：2487
  - PDF：242
  - RVT：198
  - XLSX：1
- 主要专业线索：
  - 建筑、结构、电气、给排水、消防、智能化、暖通、燃气、通用
- 工程主数据：已重置为空
- 交付标准：已重置为空
- 当前状态：真实资产已接入，但工程主数据未确认

## 本批目标

把初始化向导从“套模板生成标准”调整为“真实资产接入草案”。

M2D 不追求一次性完成真实交付标准，而是让员工能看懂：

1. 这个项目已经有哪些真实资产。
2. 资产线索能推导出哪些专业、文件类型、交付对象候选。
3. 哪些内容只是目录级线索，不能当成真实工程结论。
4. 下一步应该由项目负责人确认什么。

## 核心原则

- 目录级元数据可以辅助生成草案。
- 模板只能作为行业骨架，不得冒充真实项目结构。
- 默认不把草案直接变成“已完成交付标准”。
- 105 项目不得再次被一键套成“已就绪”。
- 不读取 PDF / Office / DWG / RVT / IFC 正文。
- 不触碰真实 NAS 文件。
- 不新增 Hermes / BIM / parser / indexing 能力。

## 开发范围

### 1. 接入评估增强

增强现有：

- `GET /api/master-data/projects/{projectId}/onboarding/assessment`
- `GET /api/master-data/projects/{projectId}/onboarding/preview`

返回更清楚的真实资产线索：

- 文件类型统计
- 扩展名统计
- 专业分布
- 相对目录线索或脱敏目录线索
- 缺 checksum / 缺专业等治理风险
- 资产线索置信度
- 不可作为正文证据的说明

### 2. 草案内容从真实资产出发

草案应至少能根据 105 当前资产生成建议：

- 专业候选：
  - 建筑、结构、给排水、暖通、电气、消防、智能化、燃气、通用
- 文件类型候选：
  - RVT 模型
  - DWG 图纸
  - PDF 图纸 / 文档
  - Excel 清单
- 交付对象候选：
  - 项目级
  - 专业级
  - 文件类型级
  - 后续可由项目负责人细化为楼栋、楼层、系统

草案必须标明：

- 来源：资产元数据 / 目录线索 / 文件类型线索 / 模板骨架
- 证据模式：catalog-only
- 置信度：HIGH / MEDIUM / LOW
- 是否需要人工确认
- 风险提示

### 3. 禁止真实项目直接套模板

对于 `asset_source` 为 `NAS_REAL*` 的真实项目：

- 前端主流程不得展示“一键套模板即完成”的语义。
- 后端 `apply-template` 或 onboarding apply 不得让真实项目未经明确真实接入确认就变成 `deliverableStandardReady=true`。
- 如保留兼容接口，必须对真实 NAS 项目返回明确提示或要求更强确认参数。

### 4. 前端体验

页面重点改：

- `ProjectInitializationPage.vue`

目标体验：

1. 员工进入 105 工程主数据页面。
2. 首屏看到“真实资产已接入，主数据未确认”。
3. 看到文件类型、专业、目录线索和风险。
4. 看到“建议草案”，而不是“模板标准已准备好”。
5. 点击后能进入部位树、节点类型、交付物标准手工确认。

### 5. 可追溯

- 草案预览不写库。
- 如本批需要写库，必须只在人工确认后执行，并写审计。
- 本批优先做只读草案增强，不强制落库。

## 禁止范围

- 不生成真实交付结论。
- 不自动锁定节点类型。
- 不自动挂接文件。
- 不自动审核。
- 不自动整改。
- 不真实打包。
- 不触碰、移动、删除、重命名 NAS 文件。
- 不读取文件正文。
- 不接入 Hermes 新能力。
- 不接入 BIM 引擎。
- 不修改 `docs/**`。

## 验收标准

1. 105 项目进入工程主数据页面后，不再展示“模板已就绪 / 标准已完成”的误导状态。
2. 105 接入评估能显示真实资产统计、专业分布、文件类型分布。
3. 草案内容能区分资产线索与模板骨架。
4. 草案项必须带证据来源、证据模式、置信度和人工确认标记。
5. 真实 NAS 项目不能一键套模板后直接变成交付标准已就绪。
6. 文档/图纸交付页面在未确认主数据时继续提示先确认工程主数据。
7. 不泄露真实 NAS 路径。
8. 不触碰真实 NAS 文件。
9. M2C/M2B/M2A/M1F/M1E/M1D/M1C 回归通过。

## 主 Agent 裁决

M2D 是 M2C 后的主线纠偏批次，用于把真实项目资产和工程主数据接起来。它优先级高于继续扩展数据管家工作区、BIM 引擎接入和客户交付准备。
