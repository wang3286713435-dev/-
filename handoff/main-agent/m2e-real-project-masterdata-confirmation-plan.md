# M2E：真实项目工程主数据人工确认与交付规则落地

## 批次定位

M2E 是 M2D 后的主线批次，目标是把 105 这类真实 NAS 项目从“资产接入 + 草案预览”推进到“人工确认后的正式工程主数据与交付规则”。

本批不是继续做模板演示，也不是自动生成完整交付结论。核心原则是：

`真实资产线索 -> 草案解释 -> 人工选择/确认 -> 生成正式主数据 -> 形成真实应交项 -> 后续交付挂接`

## 当前背景

- 105 / 503 项目的演示模板数据已软删除。
- 105 仍保留 2928 条真实文件资产。
- M2D 已提供真实资产接入评估和 catalog-only 草案预览。
- 105 当前仍应保持 `deliverableStandardReady=false`，直到员工人工确认并生成正式工程主数据。

## 本批目标

1. 让真实项目接入向导支持“人工确认草案并生成正式主数据”。
2. 让正式生成的数据带有清楚来源标记：资产线索、人工确认、行业参考模板。
3. 让 105 生成一套可解释、可编辑、不过度承诺的最小工程主数据：
   - 部位树。
   - 节点类型。
   - 交付物定义。
   - 交付物类型。
   - 目录模板或交付目录线索。
4. 生成后文档/图纸交付能出现真实应交项，而不是演示模板假应交项。
5. 员工仍可在部位树、节点类型、交付物标准页面继续人工调整。

## 设计原则

- 真实 NAS 项目不能一键套用模板并直接假装完成。
- 正式落库必须有明确人工确认参数，例如 `confirmed=true`、`confirmationMode=MANUAL_REVIEW`。
- 资产线索不足的内容只能标记为“待人工补充”或“行业参考”，不能冒充真实项目结构。
- 所有正式生成的节点/定义必须可追溯来源和风险提示。
- 生成正式标准后，才能进入文档/图纸交付的真实应交项计算。

## 建议实现

### 后端

优先在既有初始化模块内扩展，不新增大模块。

可新增或强化接口：

- `POST /api/master-data/projects/{projectId}/onboarding/confirm`

请求建议包含：

- `confirmed=true`
- `confirmationMode=MANUAL_REVIEW`
- `templateCode`
- `selectedDraftItemIds`
- `sectionStrategy`：`PROJECT_LEVEL` / `DISCIPLINE_LEVEL` / `DIRECTORY_LEVEL_MINIMAL`
- `nodeTypeStrategy`
- `deliverableStrategy`
- `riskAccepted`

响应建议包含：

- 创建的部位节点数。
- 创建的节点类型数。
- 创建的交付物定义数。
- 创建的交付物类型数。
- 仍需人工补充的事项。
- `deliverableStandardReady`。

### 前端

重点页面：

- `frontend/src/modules/master-data/pages/ProjectInitializationPage.vue`

交互建议：

1. 首屏明确显示 105 状态：真实资产已接入，工程主数据待确认。
2. 草案表格支持选择要采纳的草案项。
3. 用户选择策略后点击“确认生成工程主数据”。
4. 二次确认文案必须说明：
   - 只基于资产目录和文件元数据。
   - 不读取文件正文。
   - 生成的是可编辑的初始规则。
   - 后续仍需人工检查。
5. 完成后展示生成结果，并给出下一步入口：
   - 查看部位树。
   - 查看节点类型。
   - 查看交付物标准。
   - 进入文档交付。
   - 进入图纸交付。

## 明确禁止

- 不触碰真实 NAS 文件。
- 不读取文件正文。
- 不接入 Hermes 新能力。
- 不接入 BIM 引擎。
- 不自动挂接文件。
- 不自动审核、整改、交付。
- 不把目录元数据当正文 evidence。
- 不泄露真实 NAS 路径、`storage_uri`、SQL、token、secret。
- 不修改 `docs/**`。

## 验收标准

1. 105 在人工确认前保持 `deliverableStandardReady=false`。
2. 未确认时文档/图纸交付和交付包不出现虚假应交项。
3. 用户必须显式确认后，才生成正式工程主数据。
4. 生成后的部位树、节点类型、交付物标准能在对应页面看到。
5. 生成后的数据可解释来源，不冒充正文证据。
6. 生成后文档/图纸交付能出现真实可追踪的应交项。
7. 交付包草案基于真实生成的应交项工作。
8. 全程不触碰真实 NAS 文件，不泄露真实 NAS 路径。

## 测试策略

本批按轻量测试策略执行：

- 构建。
- 健康检查。
- M2E 专项脚本。
- M2D / M2C / M1C 必要回归。
- 核心 API 抽查。
- forbidden field 扫描。
- 不做大规模浏览器逐页点击，除非发现页面阻塞。
