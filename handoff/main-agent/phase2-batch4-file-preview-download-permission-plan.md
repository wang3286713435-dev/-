# 二期批次四规划：文件预览与下载权限分离最小闭环

更新时间：2026-05-15

## 1. 结论

二期批次四建议正式命名为：

`文件预览与下载权限分离最小闭环`

本批目标不是做模型轻量化、Office/CAD/BIM 转换、正文抽取或向量化，而是在现有“文件预览状态外壳”基础上补齐一个真正可用、可审计、可验收的文件访问闭环：

`文件详情 -> 判断预览/下载权限 -> 生成短时访问票据 -> 平台受控读取文件 -> 浏览器预览或下载 -> 审计留痕`

## 2. 与前置批次的关系

已完成能力：

1. 一期内部 BIM/CAD 资产库、NAS 元数据接管和真实 NAS 只读登记。
2. 二期批次一只读资产目录、权限证明和 Agent preview。
3. 二期批次二标准驱动交付完整率、缺失项和文件补交。
4. 二期批次三人工审核、整改闭环和 CSV 报表导出。
5. 现有接口 `GET /api/data-steward/assets/files/{fileId}/preview` 可返回预览状态，但还不是实际预览/下载闭环。

本批只补齐当前缺口：

1. 用户能看到预览状态，但不能通过平台受控打开文件。
2. 查看权限和下载权限尚未被明确拆开验证。
3. 下载入口不能暴露真实 NAS 路径，必须经过平台鉴权和审计。
4. 文件访问行为需要可被审计和回归脚本证明。

## 3. 本批范围

### 3.1 后端

建议追加 Flyway 新迁移，不修改旧迁移。

#### 权限

新增权限建议：

1. `DATA_STEWARD_FILE_PREVIEW`：文件预览。
2. `DATA_STEWARD_FILE_DOWNLOAD`：文件下载。

授权建议：

1. `PROJECT_ADMIN`：预览 + 下载。
2. `DELIVERY_ENGINEER`：预览 + 下载。
3. `PROJECT_VIEWER`：只预览，不下载。

说明：这只是二期批次四的最小权限证明，不承诺客户生产级细粒度权限体系。

#### 访问策略

新增或增强文件访问策略服务，统一判断：

1. 当前用户是否有项目访问权限。
2. 文件是否属于当前用户可访问项目。
3. 文件是否已删除、停用或隔离。
4. 文件是否有可解析的存储路径。
5. 当前用户是否具备预览或下载权限。
6. 当前格式是否支持浏览器原生预览。

#### 短时访问票据

为了避免在浏览器预览或下载时暴露 NAS 真实路径，建议采用短时访问票据：

1. `POST /api/data-steward/assets/files/{fileId}/access-tickets`
   - 请求：`action=PREVIEW|DOWNLOAD`
   - 响应：`ticketId/accessUrl/expiresAt/action/fileId/fileName/previewable/downloadable/message`
2. `GET /api/data-steward/assets/file-access/{ticket}`
   - 校验票据、用户、动作、过期时间和文件状态。
   - `PREVIEW` 使用 `inline`。
   - `DOWNLOAD` 使用 `attachment`。
   - 不返回真实 NAS 路径。

票据建议存数据库表，记录：

- `id`
- `ticket`
- `file_id`
- `project_id`
- `user_id`
- `action`
- `status`
- `expires_at`
- `used_at`
- `created_at`

#### 存储读取

本批只实现最小存储读取：

1. 支持 `nas:///Volumes/...`、`nas:///tmp/...` 这类本机挂载路径解析。
2. 只读文件，不移动、不改名、不删除。
3. 文件不存在、不可读、路径解析失败要返回清晰错误码。
4. `minio://`、对象存储、客户现场存储如暂未接入，可返回 `STORAGE_PROVIDER_UNSUPPORTED`，不得假装成功。
5. 不一次性把大文件读进内存，必须使用流式响应。

#### 预览格式

本批只开放浏览器原生预览：

1. PDF。
2. 图片：`png/jpg/jpeg/webp/gif/bmp/svg`。

Office、CAD、BIM：

1. 仍返回需要转换或不支持在线预览的状态。
2. 不做正文抽取。
3. 不做轻量化。
4. 不做构件级解析。

#### 审计

关键动作必须写审计：

1. 创建预览票据。
2. 创建下载票据。
3. 打开预览。
4. 下载文件。
5. 权限拒绝。
6. 路径失效或文件不可读。

### 3.2 前端

优先增强现有页面，不新增大模块：

1. `/data-steward/assets/{projectId}` 项目资产详情页。
2. `/data-steward/catalog` 资产目录详情抽屉。

页面应具备：

1. 清楚展示文件是否可预览、是否可下载。
2. `打开预览` 只能在可预览且有预览权限时可点。
3. `下载文件` 只能在有下载权限时可点。
4. 权限不足、文件不存在、路径不可读时展示可理解原因。
5. 不暴露真实 NAS 路径给普通用户。

### 3.3 脚本

新增专项验收脚本：

`scripts/dev/check-phase2-batch4-file-access.sh`

脚本至少覆盖：

1. 准备一个小型本机临时文件，登记为 `nas:///tmp/...` 文件资源。
2. 管理员可创建预览票据并读取内容。
3. 管理员可创建下载票据并读取内容。
4. 查看者只可预览，不可下载。
5. 跨项目用户不能创建票据。
6. 文件不存在时返回清晰错误。
7. 预览、下载、拒绝、失败动作能查到审计。
8. OpenAPI 包含新增接口。
9. 旧 `scripts/dev/check-file-preview-shell.sh` 不回归。
10. 不修改真实 NAS `/Volumes/zyzn/卓羽智能项目` 中任何文件。

## 4. 明确禁止事项

本批不做：

1. 模型轻量化在线预览。
2. 构件级解析、构件搜索、碰撞检查。
3. Office、CAD、BIM 转换服务。
4. PDF/Office/CAD/BIM 正文抽取。
5. 向量库或搜索引擎写入。
6. 企业 Agent 自动读取正文、自动审批、自动整改、自动写库。
7. 真实 NAS 文件移动、删除、改名或内容修改。
8. 客户生产级权限体系承诺。
9. 复杂统一任务中心或大规模页面重构。

## 5. 开发 Agent 交付要求

开发 agent 必须使用 Ralph Loop 推进，并把 story 拆小：

1. 权限与访问策略。
2. 短时访问票据。
3. NAS 只读流式读取。
4. 预览与下载接口。
5. 前端资产详情和目录详情入口。
6. 专项验收脚本。
7. 报告写入 `handoff/dev-agent/latest-report.md`。

完成承诺：

`<promise>PHASE2_BATCH4_FILE_ACCESS_COMPLETE</promise>`

只有当前后端构建、前端构建、健康检查、专项脚本、旧预览外壳脚本、权限拒绝、审计检查和报告全部完成后，才允许标记完成。

## 6. 测试 Agent 验收要求

测试 agent 必须重点验证：

1. 预览权限和下载权限确实分离。
2. 普通查看者不能下载。
3. 无项目权限用户不能预览或下载。
4. 平台不暴露真实 NAS 路径。
5. 文件不存在或不可读时错误清晰。
6. 访问行为写审计。
7. 旧批次一、二、三关键脚本不回归。
