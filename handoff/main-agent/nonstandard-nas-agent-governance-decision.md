# 非标准 NAS 资料 Agent 辅助治理决策

日期：2026-05-09

## 1. 决策

`98`、`95`、`99` 等非标准 NAS 目录暂不导入数据库，暂不进入正式资产库。

后续在企业 agent 接入后，将这些目录作为平台“非标准资料治理”能力处理：agent 先辅助识别，人工监控确认后，再将治理后的文件和项目导入数据库。

## 2. 暂缓目录

当前明确暂缓：

- `98-深圳口岸项目`
- `95-大鹏新区葵涌办事处葵新社区白石岗片区更新单元规划_项目`
- `95-鹏投泰润学府项目`
- `99-丰图既有建模项目`
- `99-海洋大学项目`
- 投标资料、参考资料、未知命名目录

这些目录不是“失败项”，而是治理对象。

## 3. 暂缓原因

- `98` 可能是多个口岸样板集合，不能确认是否应作为一个项目。
- `95` 和 `99` 存在重复编号，不能直接用数字编码入库。
- 投标、参考、未知目录可能不是正式项目交付资产。
- 未治理前导入会污染项目台账、权限边界、SQL View 和企业 agent 检索结果。

## 4. 后续平台能力

平台后续需要支持“非标准资料治理区”：

1. 只读发现 NAS 目录，不写正式资产。
2. agent 分析目录结构、文件名、扩展名、路径关键词和已有项目资产。
3. agent 输出治理建议：
   - 建议项目编码
   - 建议项目名称
   - 是否拆分为多个项目
   - 建议导入范围
   - 建议忽略目录
   - 风险说明
   - 置信度
4. 人工审核建议。
5. 审核通过后才允许创建项目、路径映射、扫描任务和正式资产。
6. 审核过程和导入结果必须可审计、可追溯。

## 5. 入库前置条件

非标准目录导入前必须满足：

- 已确认项目编码唯一。
- 已确认项目是否拆分。
- 已确认哪些目录是正式资产，哪些目录只是参考资料。
- 已确认文件类型规则。
- 已确认导入范围。
- 已确认负责人或审核人。

## 6. 当前结论

当前真实 NAS 入库范围到 16 个标准项目为止。

`98`、`95`、`99` 和参考/投标/未知目录暂不进入数据库。等企业 agent 接入后，再作为数据治理能力专项推进。
