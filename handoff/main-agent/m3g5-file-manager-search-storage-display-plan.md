# M3G-5 文件管理器项目全局搜索与存储展示修复计划

创建时间：2026-05-28

## 1. 批次定位

`M3G-5：文件管理器项目全局搜索与存储展示修复`

M3G-4 已经完成受控多项目小批对象化执行。但用户在真实使用中发现两个直接影响文件管理器可用性的问题：

1. 105 项目并未整体对象化，但前端没有清晰告诉用户哪些文件仍走 NAS，哪些已经对象化。
2. 文件管理器搜索只在当前目录视图内检索，用户期望搜索整个项目里的文件。

本批是 M3G 后的小修复批次，不扩大对象化范围，不执行真实迁移，不触碰 NAS 文件。

## 2. 当前事实

105 / `projectId=503` 当前不是全量对象化：

- 总文件数约 `2928`。
- 已对象化仍是小批灰度规模。
- 大多数文件仍为 `NAS_ONLY`。

因此前端必须明确展示：

- `NAS_ONLY`：仍在历史 NAS 链路。
- `OBJECT_STORED`：已有 NAS 侧 MinIO 对象副本。
- `MIGRATION_PENDING / MIGRATION_FAILED`：迁移状态。

不能让用户误以为 105 已经整体对象化。

## 3. 本批目标

- 文件管理器搜索框默认改为“项目全局搜索”。
- 搜索结果显示文件所在项目内路径，方便用户定位。
- 保留目录浏览逻辑：未搜索时仍按当前目录显示直接子文件夹和直接文件。
- 增加可选范围开关：用户需要时可限制为当前文件夹及子目录。
- 主界面不再直观暴露真实 NAS 路径或底层 `storage_uri`。
- 文件表和详情使用业务化字段：
  - 项目内路径。
  - 存储状态。
  - 访问来源：`NAS 侧 MinIO` / `历史 NAS 台账` / `迁移中` / `迁移失败`。
- 不重新递归扫描物理 NAS，不引入目录接口超时。

## 4. 允许范围

允许修改：

- `frontend/**`
- 如确实需要补只读、脱敏字段，可最小修改 `backend/delivery-data-steward/**`。
- `scripts/dev/**` 新增专项脚本。
- `handoff/dev-agent/latest-report.md`

允许的后端改动仅限：

- 返回已存在业务字段。
- 补充脱敏展示字段。
- 修正搜索参数口径。

## 5. 禁止范围

禁止：

- 执行对象化迁移。
- 扩大对象化范围。
- 移动、删除、重命名、覆盖真实 NAS 文件。
- 读取文件正文。
- 写 documents / chunks / Qdrant / OpenSearch / Hermes memory。
- Hermes 正文问答。
- BIM 引擎、parser、indexing。
- 暴露真实 NAS 路径、bucket、object key、`storage_uri`、SQL、raw row、token、secret。
- 修改 `docs/**`。

## 6. 设计口径

### A. 未搜索时

文件管理器保持 Windows 文件管理器口径：

- 左侧目录树。
- 右侧只显示当前目录直接子文件夹和直接文件。
- 不递归铺满深层文件。

### B. 输入关键词搜索时

默认切换为“项目全局搜索”：

- 不带当前目录过滤。
- 不展示当前目录直接子文件夹。
- 只展示匹配文件。
- 搜索结果必须显示“所在位置 / 项目内路径”。
- 搜索提示文案应明确：“正在整个项目中搜索”。

### C. 当前文件夹搜索

提供一个轻量开关或筛选：

- `仅当前文件夹及子目录`

开启后：

- 搜索才带当前 `directoryPath`。
- 仅查询当前目录及子目录下已登记文件。

注意：不要为这个功能递归扫描真实 NAS 物理目录。

### D. 存储展示

文件表建议新增或优化：

- `存储状态`
  - `OBJECT_STORED` 显示为：`已对象化`
  - `NAS_ONLY` 显示为：`历史 NAS`
  - `MIGRATION_PENDING` 显示为：`对象化中`
  - `MIGRATION_FAILED` 显示为：`对象化失败`
- `访问来源`
  - 已对象化：`NAS 侧 MinIO`
  - 未对象化：`历史 NAS 台账`

文件详情中：

- 默认展示项目内路径。
- 技术信息折叠区可以展示平台文件 ID、assetUuid、storage state 等。
- 不展示真实 NAS 绝对路径、bucket、object key、`storage_uri`。

## 7. 验收标准

- 105 / 503 文件管理器打开正常。
- 在根目录或任意子目录输入关键词时，默认返回整个项目匹配文件。
- 搜索结果展示文件所在项目内路径。
- 勾选“仅当前文件夹及子目录”后，搜索范围被限制。
- 清空关键词后恢复当前目录直接子项浏览。
- 文件表/详情能看出 `OBJECT_STORED` 与 `NAS_ONLY` 差异。
- 主界面不暴露真实 NAS 路径、bucket、object key、`storage_uri`。
- 不触发真实对象化迁移。
- 不修改真实 NAS 文件。
- M3G-4 / M3G-3 / M3E / file-access 回归通过。
