# M2B：受控 NAS 写操作真实项目灰度试运行与安全开关

## 1. 批次定位

`M2B` 是 M2A 收口后的真实项目灰度试运行批次。

M2B 不新增完整 NAS CRUD，不做永久删除，不做批量不可逆操作。它的目标是把 M2A 已完成的受控 NAS 写操作能力，放到一个低风险真实项目或项目内试运行目录中，验证真实员工、真实权限、真实目录、真实回滚流程。

## 2. 启动前提

M2A 已收口，并已修复真实使用缺陷：项目工作台路由项目可直接执行 NAS 操作，不要求 JWT 当前项目先切换。

M2A 专项脚本当前结果：`PASS=21 FAIL=0`。

## 3. 本批目标

完成真实项目灰度试运行底座：

- 平台可以为某个项目开启 / 关闭 NAS 写操作灰度。
- 平台可以限制灰度操作目录范围。
- 平台可以限制可操作账号和角色。
- 平台可以在页面明确展示“当前处于真实 NAS 灰度试运行”。
- 平台可以记录灰度操作、回滚动作、失败原因和审计。
- 员工只能在被授权的项目和目录内执行 M2A 已允许的操作。

## 4. 允许范围

允许：

- 新增或复用安全开关配置。
- 限制真实项目可写目录，例如 `平台试运行区` 或管理员指定的项目内相对目录。
- 在文件管理页增加灰度状态提示、可写范围提示和禁用原因。
- 对 M2A 写接口增加灰度开关与目录范围校验。
- 增加 M2B 专项脚本，使用真实项目的安全试运行目录或可配置测试目录验证。

## 5. 禁止范围

禁止：

- 永久物理删除。
- 跨项目移动。
- 批量删除 / 批量移动 / 批量重命名。
- Agent / Hermes 触发 NAS 写操作。
- 前端传入或展示真实 NAS 绝对路径。
- 文件正文读取。
- BIM 轻量化、构件解析、索引。
- 修改 UX1 前端体验专项范围。
- 为某个项目写死逻辑。

## 6. 默认灰度策略

若用户没有提供真实试点项目和目录，开发 agent 应采用安全默认策略：

- 后端支持灰度开关与目录范围，但默认关闭。
- 专项脚本使用临时目录或显式配置的安全目录。
- 真实项目页面默认只显示“未开启真实 NAS 写入灰度”。
- 未开启灰度时，M2A 写接口返回清晰业务错误，不执行真实文件操作。

## 7. 建议能力

- `NAS_WRITE_TRIAL_ENABLED`：项目级灰度开关。
- `allowedRelativeRoots`：项目内允许写入的相对目录白名单。
- `allowedRoleCodes`：允许执行灰度操作的项目角色。
- `trialModeNotice`：前端展示的试运行提示。
- `operationPrecheck`：执行前返回是否允许、禁止原因和可操作范围。

具体实现方式可结合现有表结构，但必须保持服务端强校验。

## 8. 验收红线

P0：

- 未开启灰度也能写真实 NAS。
- 可以写出允许目录范围。
- 查看者或未授权用户可写。
- 永久删除真实文件。
- 响应泄露真实 NAS 路径。
- Agent / Hermes 触发写操作。

## 9. 收口条件

- 后端构建通过。
- 前端构建通过。
- M2B 专项脚本通过。
- M2A / M1F / M1E / M1D / M1C 回归通过。
- 未开启灰度时写操作被拒绝。
- 开启灰度后仅允许在指定目录执行 M2A 允许的操作。
- 页面清楚展示灰度状态和风险提示。
