# G4：真实项目交付闭环试运行与问题修补

更新时间：2026-05-20

## 1. 批次定位

`G4：真实项目交付闭环试运行与问题修补` 是 G3 收口后的真实项目试运行批次。

G4 不新增大模块，不进入 8B / 8C / 9A，不启动真实 BIM 轻量化，不开放真实 NAS 增删改查。它的目标是把当前已经做出的能力放到真实 NAS 项目里跑一遍，找出阻塞真实员工使用的 P0/P1/P2，并做小范围修补。

G4 的核心问题不是“平台还能加什么”，而是：

- 真实项目能不能顺利从资产台账进入交付工作台。
- 员工能不能在 Hermes 帮助下理解工程主数据、交付缺失项和下一步动作。
- 现有批量挂接、审核、整改、导出预检查是否能组成一条可用闭环。
- 页面、提示、权限、审计和数据刷新是否足够支撑真实试运行。

## 2. 前置状态

已收口能力：

- 一期真实 NAS 资产接管与后端数据治理。
- 二期 1 / 2 / 3 / 4R / 5A / 5A.1 / 5B / 6A / 6B / 7A / 7B / 8A。
- G1：Agent 引导式交付治理 MVP。

- 8C：真实 BIM 引擎小样本接入。
- 9A：客户部署 / 产品化交付相关批次。

## 3. 试运行样本

必须覆盖：

1. `105` 项目作为主样本。
2. 至少另一个已接管的真实 NAS 项目作为泛化验证样本。

要求：

- 禁止为 105 写死逻辑。
- 修复必须对真实 NAS 项目通用。
- 测试数据如需创建，应使用可识别的测试标记，并在报告里说明是否清理。

## 4. 本批主链路

G4 需要围绕以下闭环做真实试运行：

1. 从 `/data-steward/assets` 进入真实 NAS 项目。
2. 查看项目接入状态和项目资产概况。
3. 进入项目工作台。
4. 打开 Hermes，询问当前项目如何进入数字化交付。
5. 使用 Hermes Action Center 生成工程主数据补齐计划。
6. 检查工程主数据：
   - 部位树。
   - 节点类型。
   - 交付物标准。
7. 使用 Hermes 生成交付缺失项补交 / 文件挂接推荐方案。
8. 人工确认后调用现有推荐挂接能力。
9. 查看文档交付 / 图纸交付完整率变化。
10. 对需要审核的交付项执行审核 / 驳回 / 整改闭环。
11. 执行交付包导出预检查 dry-run。
12. 查看审计记录，确认关键动作可追踪。

## 5. 允许修补范围

G4 允许处理以下小范围问题：

- 真实项目入口、跳转、项目上下文丢失。
- Hermes 提示不清楚、Action Center 状态不清楚。
- 工程主数据页面缺少从真实项目回来的入口。
- 推荐挂接后页面不刷新或结果表达不清楚。
- 文档 / 图纸交付页面状态与推荐挂接结果不一致。
- 审核 / 整改 / 导出预检查入口不清楚。
- 空状态、失败提示、权限提示、Missing Evidence 提示不适合普通员工。
- 试运行专项脚本缺失或无法复跑。
- 不影响底层模型的小型只读聚合或前端展示修复。

## 6. 明确禁止事项

G4 严禁：

- 进入 8B / 8C / 9A。
- 真实 BIM 轻量化、真实模型转换、构件级解析。
- PDF / Office / DWG / RVT / IFC 正文抽取。
- selective indexing。
- 写 Hermes memory。
- 写 OpenSearch / Qdrant / MinIO documents/chunks。
- 真实 NAS 文件上传、移动、删除、重命名。
- Agent 自动审批、自动整改、自动挂接。
- Hermes 绕过平台权限或直接写数据库。
- 把 catalog metadata 冒充正文 evidence。
- 返回真实 NAS 绝对路径、raw row、SQL、token、secret、password。
- 只对 105 做硬编码。

## 7. 验收标准

G4 至少满足：

1. 105 项目可以跑通真实项目交付闭环试运行。
2. 至少另一个真实 NAS 项目可以跑通同类抽查。
3. Hermes 能解释当前项目接入状态、工程主数据状态、交付缺失项和下一步动作。
4. Hermes 能生成工程主数据补齐计划和交付缺失项补交推荐方案。
5. 未人工确认时不能执行写动作。
6. 人工确认后推荐挂接能通过平台既有能力执行。
7. 文档 / 图纸交付完整率和缺失项状态能刷新。
8. 审核、整改、导出预检查可进入并保持项目上下文。
9. 关键动作有审计。
10. 正文 / DWG / RVT / BIM 构件类问题仍返回 Missing Evidence。
11. 不泄露真实 NAS 路径、raw row、SQL、token、secret、password。
12. G3 / G2 / G1 / 8A 回归不破坏。
